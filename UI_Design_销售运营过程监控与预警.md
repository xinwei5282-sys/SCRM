# 销售运营过程监控与智能预警 UI 设计方案

> 设计基础：`.claude/design-foundation.md`
> 设计语言：Nova 紧凑布局
> 版本：v1.1 | 日期：2026-04-13
> PRD 对应：[PRD_销售运营过程监控与预警.md](PRD_销售运营过程监控与预警.md)

---

## 1. 设计拆分

本需求拆成两个终端：

1. `CRM 后台经营监控报表`
2. `企微销售预警工作台`

两端服务对象不同，因此不应混成一个页面。

---

## 2. CRM 后台经营监控报表

### 2.1 目标用户

- 销售主管 / 负责人
- 运营负责人

### 2.2 页面定位

后台页负责：

- 看经营过程
- 看漏斗断点
- 看销售差异
- 看渠道质量
- 看销售对预警的处理情况

不是给一线做高频待办入口。

### 2.3 页面结构

```text
┌──────────────────────────────────────────────────────────────┐
│ CRM 后台 Header / 全局导航                                   │
├──────────────────────────────────────────────────────────────┤
│ 页面标题 + 导出 + 查看处理监控                               │
├──────────────────────────────────────────────────────────────┤
│ 经营摘要 Hero                                                 │
├──────────────────────────────────────────────────────────────┤
│ Filter Bar                                                   │
├──────────────────────────────────────────────────────────────┤
│ KPI Row                                                      │
├───────────────────────────────┬──────────────────────────────┤
│ 漏斗与重点假设 / 销售对比 / 渠道质量 │ 销售处理监控            │
├──────────────────────────────────────────────────────────────┤
│ 客户明细与推荐动作                                           │
└──────────────────────────────────────────────────────────────┘
```

### 2.4 关键模块

- 经营摘要：用一句话讲清当前主要断点
- KPI 卡片：线索、双向好友率、进群率、问卷率
- 漏斗区：完整经营链路
- 假设区：本期待验证假设
- 销售过程对比：看承接差异
- 渠道质量：看来源质量
- 销售处理监控：看预警是否被及时处理和关闭
- 客户明细：支撑下钻与复盘

---

## 3. 企微销售预警工作台

### 3.1 目标用户

- 一线销售

### 3.2 页面定位

企微工作台负责：

- 接收与自己有关的预警
- 快速理解提醒原因
- 立即执行处理动作
- 回传处理状态

不承担团队经营分析。

### 3.3 页面结构

```text
┌──────────────────────────────┐
│ Header：我的待处理预警        │
├──────────────────────────────┤
│ 顶部摘要：待处理 / 超时 / 完成 │
├──────────────────────────────┤
│ 分类筛选 Chips                │
├──────────────────────────────┤
│ 预警卡片列表                  │
│ - 客户名                      │
│ - 规则类型                    │
│ - 触发原因                    │
│ - 建议动作                    │
│ - 快捷按钮                    │
├──────────────────────────────┤
│ Footer：历史入口 / 补充说明    │
└──────────────────────────────┘
```

### 3.4 设计原则

- 一条提醒必须在一屏内看懂
- 每张卡片必须给出下一步动作
- 操作优先，不做复杂图表
- 已处理状态要明显，避免重复提醒

### 3.5 关键模块

- 我的待处理预警
- 分类筛选
- 单条卡片处理
- 已处理记录
- 快速入口：查看客户 / 查看问卷 / 查看原话 / 立即联系

---

## 4. 状态与映射

### 4.1 后台状态

- 待处理
- 已查看
- 已处理
- 已关闭
- 已超时
- 忽略

### 4.2 企微状态展示

- `P1`：红色高优先级
- `P2`：橙色提醒
- `已完成`：绿色完成态
