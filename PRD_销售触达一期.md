## 1. 文档概述

### 1.1 产品背景

当前系统已经具备客户标签与模板库能力，但尚未形成可执行的销售触达链路。销售和运营在实际工作中仍需要依赖人工筛人、手动组织内容、线下跟踪执行结果，存在以下问题：

- 无统一触达入口，任务创建与执行分散
- 无前端可配置的人群圈选能力
- 无基础防骚扰机制，容易重复打扰客户
- 无结果回收与复盘页面，无法评估触达效果

考虑到一期目标是尽快让触达能力可用，本期通过单页表单完成触达任务创建，支持条件圈选人群、手动填写内容、选择发送通道，将销售触达主流程打通。

### 1.2 产品目标

- 提供灵活的人群条件圈选能力，支持组合多维字段筛选目标客户
- 支持企微官方群发与星云两个发送通道
- 系统按设置规则自动完成防骚扰排除（拉黑/删除客户过滤）
- 支持销售/运营手动填写触达内容并发起任务
- 支持查看任务状态、执行结果与客户回复情况
- 为后续模板联动、自动旅程预留数据结构

### 1.3 核心指标

| 指标 | 目标值 |
|------|--------|
| 单个触达任务创建时长 | < 3 分钟 |
| 任务命中人数预览响应时间 | < 5 秒 |
| 运营可独立完成触达任务创建 | 100% |
| 任务结果可追溯率 | 100% |

### 1.4 范围

**In-Scope（本期）**：

- 触达任务创建（条件圈选人群 + 手动填写内容 + 发送通道选择）
- 草稿保存与任务修改
- 立即发送 / 定时发送
- 任务列表与状态管理
- 结果详情与基础效果统计
- 基础防骚扰拦截（拉黑/删除客户过滤，固定开启）

**Out-of-Scope（本期不做）**：

- 拦截记录独立页面（已从一期删除，后续排期）
- 人群规则配置器（前台可视化配置）
- 模板变量渲染与模板联动
- 自动化旅程 / SOP 编排
- A/B 测试
- 审批流
- 智能推荐发送时间
- 复杂转化归因

## 3. 整体方案

### 3.1 方案原则

一期采用"人群条件圈选 + 手填内容 + 通道选择 + 任务执行 + 结果回收"的方案，不做过度平台化设计。

整体链路如下：

`填写基础信息 → 圈选目标人群 → 防骚扰排除 → 手动填写内容 → 选择发送时机 → 创建任务 → 执行发送 → 回收结果 → 查看效果`

### 3.2 模块结构

本期建设 3 个核心模块：

1. `触达任务创建`（含草稿保存与任务修改）
2. `触达任务列表`
3. `触达结果详情`

### 3.3 设计约束

- 人群圈选通过前端条件编辑器完成，运算符和字段值由系统枚举提供
- 推送内容不依赖模板渲染，由用户手动填写
- 任务执行时必须保存目标客户快照，避免结果复盘失真
- 防骚扰规则一期固定（拉黑/删除客户过滤），不开放策略后台，始终开启

## 4. 功能需求

### 4.1 触达任务创建

#### 4.1.1 页面目标

让销售或运营在单页表单内完成触达任务的完整配置，支持新建、修改与草稿保存。

#### 4.1.2 页面标题与结构

- 新建模式标题：**新建触达任务**
- 只读模式标题：**触达任务详情**

单页表单，自上而下分为 4 个区块，区块可滚动，footer 固定在底部：

1. **基础设置** — 任务名称、所属鱼丸、发送通道
2. **目标人群** — 条件编辑器
3. **推送内容与发送方式** — 文本、富媒体附件、发送方式（同一区块）
4. **防骚扰** — 拉黑/删除客户过滤（固定开启，不可关闭）

顶部非固定，跟随内容区滚动；页面最大高度为一屏，内容超出仅在中间内容区滚动。

#### 4.1.3 基础设置字段

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 任务名称 | 手动填写，建议含活动名+日期 | 是 |
| 所属鱼丸 | 全部鱼丸 / 部分鱼丸 | 是 |
| 发送通道 | 企微官方群发（合规）/ 星云（谨慎使用），卡片单选 | 是 |

**所属鱼丸交互规则**：
- 支持「全部鱼丸 / 部分鱼丸」切换
- 选择部分鱼丸时，点击输入区域打开企微组织架构弹层
- 弹层同时展示部门与企微成员；已选鱼丸以 tag 展示，支持逐个删除
- 删除 tag 后同步取消组织架构内对应勾选

#### 4.1.4 目标人群字段

条件编辑器，支持多条件组合，条件间支持且 / 或 / 非连接。

**可选字段穷举**：

| 字段 | 可选运算符 | 可选值 | 备注 |
|------|-----------|--------|------|
| 企微标签 | 满足任一 / 同时满足 / 全部不满足 | 自由输入标签名（支持多个，以 chip 展示） | — |
| 客户等级 | 等于 / 不等于 | S / A / B / C | 下拉单选 |
| 是否领卡 | 等于（固定，不可更改） | 是 / 否 | 下拉单选 |
| 是否是官网会员 | 等于（固定，不可更改） | 是 / 否 | 下拉单选 |
| 是否加入企微群聊 | 等于（固定，不可更改） | 是 / 否 | 下拉单选 |
| 是否观看直播 | 等于（固定，不可更改） | 是 / 否 | 下拉单选 |
| 是否分销 | 等于（固定，不可更改） | 是 / 否 | 下拉单选 |
| 体验营期数 | 等于 / 不等于 | 1期 / 2期 / 3期 / 4期 / 5期 | 下拉单选 |
| 首次加入时间 | 等于 / 开始于 / 结束于 / 为空 / 不为空 | 日期输入（格式 YYYY-MM-DD）；选「为空 / 不为空」时值区隐藏 | — |
| 添加好友时间 | 等于 / 开始于 / 结束于 / 为空 / 不为空 | 日期输入（格式 YYYY-MM-DD）；选「为空 / 不为空」时值区隐藏 | — |
| 所属销售 | 等于 / 不等于 / 包含 | 销售人员下拉（系统成员列表） | — |
| 分销人 | 等于 / 不等于 / 包含 | 销售人员下拉（系统成员列表） | — |
| 浏览文章时长 | 大于 / 小于 / 大于等于 / 小于等于 / 等于 / 不等于 | 数值输入（单位：分钟） | — |

**人群 footer**：展示预计命中人数与「刷新估算」按钮，始终固定在人群区块底部。

#### 4.1.5 推送内容与发送方式（同一区块）

**推送内容**：

| 字段 | 说明 |
|------|------|
| 推送文本 | 必填，手动填写，建议 1000 字以内 |
| 快捷变量 | 文本区上方入口，可插入「昵称 / 备注」占位符 |
| 富媒体附件 | 上传图片 / 上传视频 / 添加链接 / 添加小程序 / 添加视频号，最多 9 个附件 |

内容快照在提交时完整存档到任务详情页，不可事后修改。

**发送方式**（必填，与推送内容同区块）：

| 选项 | 说明 |
|------|------|
| 立即发送 | 提交后立即执行 |
| 定时发送 | 必填发送时间；限 09:00–21:00；超出范围显示错误提示 |

#### 4.1.6 防骚扰设置

名称：**拉黑 / 删除客户过滤**，描述：「已将销售删除或拉黑的客户始终排除（不可关闭）」。始终开启，不可关闭，无需用户配置。

#### 4.1.7 footer

Footer 固定在页面底部，分左右两侧：

- **左侧**：预计命中人数（如「预计命中：1,248 人」）+ 「刷新估算」按钮
- **右侧**：`取消`（返回列表，不保存）/ `保存草稿`（草稿状态）/ `保 存`（校验通过后提交执行）

推送内容为空时，「保 存」不可点击。

#### 4.1.8 任务详情（只读模式）

列表「详情」入口复用创建页，切换为只读态：

- 所有输入控件禁用
- footer「保存草稿」与「保存」按钮同步禁用
- 顶部返回按钮文案改为「返回列表」（编辑态显示「取消」）
- 不新增独立路由，通过模式标记切换

#### 4.1.9 校验规则

| 字段 | 校验规则 | 提示文案 |
|------|----------|----------|
| 任务名称 | 非空 | 请输入任务名称 |
| 发送通道 | 必选 | 请选择发送通道 |
| 所属鱼丸 | 部分鱼丸时至少选 1 个 | 请选择所属鱼丸 |
| 推送内容 | 非空 | 请输入推送内容 |
| 发送时间 | 定时发送时必填 | 请选择发送时间 |
| 发送时间 | 限 09:00–21:00 | 发送时间超出允许范围（09:00 – 21:00），请重新设置 |

#### 4.1.10 异常场景

| 场景 | 处理方式 |
|------|----------|
| 命中人数为 0 | 允许保存草稿，不允许提交执行 |
| 人数估算失败 | 提示"人群预览失败，请稍后重试" |

### 4.2 触达任务列表

#### 4.2.1 页面目标

展示所有触达任务，支持按状态 tab 筛选并进入对应操作。

#### 4.2.2 列表字段

| 字段 | 说明 |
|------|------|
| 任务名称 | 可点击进入任务详情 |
| 创建人 | 任务创建者 |
| 发送方式 | 立即发送 / 定时发送 |
| 计划时间 | 定时任务展示 |
| 状态 | 草稿 / 待执行 / 执行中 / 已完成 / 部分失败 |
| 预计触达 | 命中并通过防骚扰的目标人数 |
| 实际触达 | 实际进入执行链路的人数 |
| 回复人数 | 产生回复的人数 |
| 回复率 | 回复人数 / 实际触达人数 |

#### 4.2.3 筛选区

- 搜索栏：任务名称关键词搜索 + 重置按钮
- 状态 Tab（搜索栏下方）：全部 / 草稿 / 待执行 / 执行中 / 已完成 / 部分失败

#### 4.2.4 操作项（按状态区分）

| 状态 | 可用操作 |
|------|----------|
| 草稿 | 修改 / 删除 |
| 待执行 | 修改 / 复制 / 删除 |
| 执行中 / 已完成 / 部分失败 | 数据 / 详情 / 复制 |

#### 4.2.5 分页

底部分页区：上一页 / 页码 / 下一页 / 每页条数（10 / 20 / 50 条/页）。

#### 4.2.6 空态与异常态

| 场景 | 展示 |
|------|------|
| 无任务数据 | 空态插图 + 「去创建任务」按钮 |
| 筛选后无结果 | 「暂无符合条件的任务」 |
| 列表加载失败 | 失败提示 + 「重新加载」按钮 |

## 5. 防骚扰机制

### 5.1 设计目标

在任务执行前对客户进行基础防骚扰校验，降低重复触达和负向体验风险。

### 5.2 一期固定规则

一期仅执行以下硬拦截规则（固定，不开放配置）：

- 已拉黑 / 已删除好友

### 5.3 规则输出

系统需支持：

- 返回客户是否可触达
- 任务提交时展示排除人数
- 任务详情中保存被拦截客户的快照记录（排除原因可在客户明细中查看）

## 6. 状态流转

### 6.1 任务状态

- 草稿
- 待执行
- 执行中
- 已完成
- 部分失败
- 执行失败
- 已取消

### 6.2 状态流转规则

- 保存草稿 → `草稿`
- 提交立即发送 → `待执行` → `执行中`
- 提交定时发送 → `待执行`（到时触发）→ `执行中`
- 执行结束 → `已完成 / 部分失败 / 执行失败`
- 草稿 / 待执行时可取消 → `已取消`

## 7. 数据对象

### 7.1 `touch_task`

任务主表，记录任务基础信息。

核心字段：

- `id`
- `task_name`
- `channel_code`（发送通道：qiwei_official / xingyun）
- `yupan_scope`（所属鱼丸范围）
- `audience_conditions`（人群条件 JSON）
- `content`
- `send_type`
- `schedule_time`
- `status`
- `creator_id`
- `created_at`
- `updated_at`

### 7.2 `touch_task_target`

任务目标客户快照表，保存任务创建时的目标客户结果。

核心字段：

- `id`
- `task_id`
- `customer_id`
- `owner_id`
- `is_blocked`
- `block_reason`
- `created_at`

### 7.3 `touch_task_execution`

任务执行记录表。

核心字段：

- `id`
- `task_id`
- `executor_id`
- `customer_id`
- `execution_status`
- `executed_at`
- `failure_reason`

### 7.4 `touch_task_result`

任务结果回流表。

核心字段：

- `id`
- `task_id`
- `customer_id`
- `send_status`
- `reply_status`
- `reply_time`
- `result_code`
- `updated_at`

### 7.5 `touch_task_log`

任务日志表。

核心字段：

- `id`
- `task_id`
- `log_type`
- `log_content`
- `operator_id`
- `created_at`

> 关键要求：`touch_task_target` 必须为快照表，不能在结果页实时按当前客户状态重新计算。

## 9. 技术考虑

### 9.1 执行链路

- 触达任务提交时先计算命中客户，再执行防骚扰排除
- 任务提交后生成客户快照与执行记录
- 发送结果和回复结果需支持异步回流

### 9.2 平台约束

- 一期需兼容当前企微官方数发与星云两条发送链路
- 若发送通道存在官方限制，则系统侧以任务管理、执行记录、结果回收为主，不在 PRD 中承诺超出当前通道能力的自动发送能力

### 9.3 可扩展性预留

- 人群条件字段后续可扩充，audience_conditions 使用 JSON 存储以支持字段扩展
- 手动填写内容后续可接入模板库与变量渲染
- 当前结果表结构需支持后续增加点击、转化等指标

## 10. 风险与依赖

### 10.1 依赖项

- 客户基础信息与鱼丸负责人数据可用
- 标签数据与客户属性字段与人群条件编辑器字段对齐
- 两个发送通道可以返回基础执行结果
- 回复结果可被系统回收

### 10.2 风险点

| 风险 | 说明 | 应对方式 |
|------|------|----------|
| 人群条件组合过复杂 | 复杂条件计算耗时可能超过预期 | 增加计算超时保护和 loading 状态 |
| 发送链路能力不一致 | 企微与星云返回结果粒度不同 | 结果字段按最低公共能力设计 |
| 手填内容质量不稳定 | 不同成员内容质量差异大 | 一期通过操作说明约束 |

## 13. 执行时序

### 13.1 核心参与方

- `用户端`：销售 / 销售运营
- `触达任务服务`：负责任务创建、状态流转、日志记录
- `人群计算服务`：负责按条件计算命中客户
- `防骚扰规则服务`：负责客户排除与拦截原因计算
- `发送执行服务`：负责将任务下发到实际发送链路
- `结果回流服务`：负责接收发送结果与回复结果

### 13.2 预览命中人数时序

```text
用户端
  -> 触达任务服务：提交 audienceConditions、yupanScope
触达任务服务
  -> 人群计算服务：按条件计算命中客户
人群计算服务
  -> 触达任务服务：返回原始命中客户列表
触达任务服务
  -> 防骚扰规则服务：执行拦截校验
防骚扰规则服务
  -> 触达任务服务：返回可触达客户与拦截数
触达任务服务
  -> 用户端：返回 hitCount、blockedCount
```

### 13.3 创建任务时序

```text
用户端
  -> 触达任务服务：提交任务表单
触达任务服务
  -> 校验服务：基础字段校验
触达任务服务
  -> 人群计算服务：重新计算命中客户
触达任务服务
  -> 防骚扰规则服务：重新执行拦截校验
触达任务服务
  -> 数据库：写入 touch_task
触达任务服务
  -> 数据库：写入 touch_task_target 快照
触达任务服务
  -> 数据库：写入 touch_task_log
触达任务服务
  -> 用户端：返回 taskId、taskStatus
```

**草稿与提交差异**：

| 动作 | 是否重算人群 | 是否保存客户快照 | 任务状态 |
|------|--------------|------------------|----------|
| 保存草稿 | 否 | 否 | 草稿 |
| 提交执行 | 是 | 是 | 待执行 |

### 13.4 执行与回流时序

适用于立即发送（定时发送在到达时间后进入同一链路）：

```text
触达任务服务
  -> 发送执行服务：下发任务
发送执行服务
  -> 数据库：写入 touch_task_execution（待执行）
发送执行服务
  -> 实际发送链路：触发发送
实际发送链路
  -> 发送执行服务：返回发送受理结果
发送执行服务
  -> 数据库：更新 touch_task_execution
发送执行服务
  -> 触达任务服务：返回任务执行进度
触达任务服务
  -> 数据库：更新 touch_task 状态
```

### 13.5 结果回流时序

```text
发送链路 / 外部回调
  -> 结果回流服务：提交发送结果或回复结果
结果回流服务
  -> 数据库：更新 touch_task_result
结果回流服务
  -> 触达任务服务：触发任务聚合计算
触达任务服务
  -> 数据库：更新任务汇总指标与任务状态
触达任务服务
  -> 数据库：写入 touch_task_log
```

**聚合逻辑**：

- 全部成功 → `已完成`
- 部分成功部分失败 → `部分失败`
- 全部失败 → `执行失败`
- 回复结果不改变任务完成状态，但会更新回复指标

### 13.6 日志记录要求

以下关键节点必须写入 `touch_task_log`：

- 创建任务 / 保存草稿
- 提交执行
- 人群计算完成
- 防骚扰排除完成
- 开始执行
- 执行完成 / 失败
- 取消任务
- 结果回流更新

## 14. 状态机细化

### 14.1 任务状态机

```text
草稿
  -> 待执行      （提交执行）
  -> 已取消      （取消）

待执行
  -> 执行中      （开始执行）
  -> 已取消      （取消）

执行中
  -> 已完成      （全部成功）
  -> 部分失败    （部分成功部分失败）
  -> 执行失败    （全部失败）

已完成 / 部分失败 / 执行失败 / 已取消
  -> 终态
```

### 14.2 客户级执行状态机

```text
待执行
  -> 已拦截      （命中防骚扰规则）
  -> 发送中      （进入发送链路）

发送中
  -> 成功        （发送成功）
  -> 失败        （发送失败）

成功
  -> 已回复      （收到客户回复）
```
