## 1. 文档概述

### 1.1 背景与目标

We're building **新手任务引导系统** for **生财有术新会员** to solve **新用户不知道从哪里开始使用平台**, which will result in **新用户 7 日内激活率提升、流失率降低**。

**业务目标**：

- **提升新用户激活率**：引导完成关键操作（绑定账号、添加运营人员、完善个人信息），降低新用户 7 日内流失率
- **完善用户画像**：通过新人登记表收集用户基础信息，支撑后续精细化运营

### 1.2 范围

**包含**：
- 移动端：新手任务弹窗 + 任务列表页（3 个任务）
- PC 端：新手任务弹窗（弹窗内完成，2 个任务）+ 悬浮提醒条
- 任务完成检测（OAuth 回调、企微回调、表单提交回调）
- 跨端状态同步
- 数据埋点

**不包含**：
- 任务完成奖励积分/勋章系统
- 后台自定义任务配置界面（v1 写死任务列表）
- 任务排行榜/社交分享
- 推送通知提醒未完成任务
- A/B 测试框架（v2 规划）

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 生财有术新会员 | 25-40 岁，副业探索者/自媒体从业者/电商卖家/职场人，互联网熟练度中等偏上，熟悉微信生态 | 快速了解平台价值，找到与自己相关的内容和人 |

### 2.2 问题陈述

**I am:** 刚付费加入生财有术的新会员，对平台功能和社群生态完全陌生
**Trying to:** 快速了解平台价值和使用方法，尽快融入社群并获得实际收益
**But:** 面对一个全新平台，不知道从哪里开始，也不知道有哪些核心功能值得立即使用
**Because:** 平台缺少结构化的新手引导，用户需要自行探索
**Which makes me feel:** 焦虑和迷茫——已经付费了却不知道如何获取价值，担心错过重要功能

**结论**：生财有术新会员需要一套结构化的新手任务引导，帮助他们在首次登录后的 10 分钟内完成关键操作（绑定账号、连接运营人员、完善信息），从而快速感知平台价值并融入社群。

### 2.3 用户故事

- **作为**新用户，**我希望**首次访问首页时看到新手任务引导弹窗，**以便**知道下一步该做什么
- **作为**新用户，**我希望**在任务列表中看到所有任务及进度，**以便**有清晰的完成目标
- **作为**新用户，**我希望**点击按钮就能完成对应操作，**以便**快速上手平台核心功能
- **作为**新用户，**我希望**完成全部任务后获得庆祝反馈，**以便**产生成就感和归属感
- **作为**已在其他渠道完成过操作的用户，**我希望**系统自动识别，**以便**不需要重复操作

### 2.4 使用场景

**场景一：首次登录引导**
新用户注册后首次访问首页 → 600ms 后自动弹出新手任务弹窗 → 点击"去完成" → 移动端跳转任务列表页 / PC 端在弹窗内展示任务 → 逐个完成任务 → 全部完成后显示庆祝界面

**场景二：断点恢复**
用户完成部分任务后离开 → 再次访问首页 → 弹窗再次弹出（本次会话首次）/ 通过悬浮提醒条（PC 端）找到入口 → 继续完成剩余任务

**场景三：跨端切换**
用户在移动端完成"绑定星球账号" → 切换到 PC 端 → 打开弹窗时自动拉取最新状态 → 仅显示 PC 端需要完成的任务，已完成的任务同步标记

**场景四：老用户补做**
功能上线前已注册的用户，若未完成对应操作 → 首次访问时同样触发新手任务 → 已有操作（如已添加鱼丸微信）自动标记为已完成

## 3. 功能需求

### 3.1 功能清单

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|-------|-------|-----|
| 任务展示 | 任务列表展示 | P0 | 展示所有新手任务，包括已完成和未完成的任务 |
| 任务展示 | 任务进度提示 | P0 | 在任务页面/弹窗顶部显示任务完成进度条 |

| 任务执行 | 跨端状态同步 | P0 | 任务状态存储在服务端，客户端通过 API 获取最新状态 |
| 用户引导 | 首页任务提醒弹窗 | P0 | 首页自动弹出新手任务弹窗，引导用户完成任务 |

- P0：核心功能，MVP 必须包含（不超过 5 个模块）
- P1：重要功能，第一版强烈建议包含
- P2：增强功能，后续迭代考虑

### 3.2 功能详细说明

#### 3.2.1 任务列表展示与进度

**Epic Hypothesis**：
If we 在首页提供结构化的新手任务列表和实时进度条 for 新注册会员 then we will 将新手任务完成率提升至 60% 以上（上线 30 天内）

**功能描述**：
展示所有新手任务的列表视图，顶部显示任务完成进度条（X/Y 已完成），每个任务卡片包含名称、描述和操作按钮。

**PC 端与移动端任务差异**：

| 任务名称 | 移动端 | PC 端 | 说明 |
|----------|--------|-------|------|
| 绑定星球账号 | 显示 | 不显示 | 星球账号绑定为移动端专属操作 |
| 扫码添加鱼丸 | 显示 | 显示 | 双端均需完成，PC 端展示二维码供手机扫描 |
| 填写新人登记表 | 显示 | 显示 | 双端均需完成，复用现有新人登记表功能 |

> PC 端共 2 个任务，移动端共 3 个任务。任务完成状态双端同步。

**交互逻辑**：
1. 移动端：用户进入任务列表页 → 查看任务和进度 → 点击操作按钮 → 跳转对应操作页面 → 完成后返回自动刷新状态
2. PC 端：用户在弹窗内查看任务和进度 → 点击操作按钮 → 弹窗内弹出操作组件（如二维码） → 完成后弹窗内实时更新状态

**业务规则**：
- 移动端显示 3 个任务卡片，PC 端显示 2 个任务卡片
- 已完成的任务按钮切换为灰色"已完成"态，不可再次点击
- 首次加载超过 500ms 时显示骨架屏占位
- 接口返回失败时显示"加载失败，点击重试"

**验收标准**：
- Given 新用户首次进入任务页面 When 页面加载完成（< 1.5s） Then 移动端显示 3 个任务卡片，PC 端显示 2 个任务卡片，顶部显示进度条
- Given 页面加载中且接口响应 > 500ms When 等待期间 Then 显示骨架屏占位
- Given 接口返回失败 When 页面加载完成 Then 显示"加载失败，点击重试"，点击后重新请求

**异常处理**：
- 网络请求失败：列表区显示"加载失败，点击重试"，居中展示
- 部分任务状态获取失败：显示已获取的任务，失败的任务显示加载中态

#### 3.2.2 任务完成检测与状态管理

**Epic Hypothesis**：
If we 通过 OAuth 回调、企微回调和表单回调自动检测任务完成 for 正在完成新手任务的用户 then we will 实现任务状态在 3s 内自动更新，无需用户手动刷新

**功能描述**：
自动检测每个任务的完成情况，实时更新任务状态和进度条。支持跨端同步。

**每个任务的完成检测规则**：

| 任务 | 完成触发条件 | 检测方式 | 超时处理 |
|------|------------|---------|---------|
| 绑定星球账号 | OAuth 授权成功，服务端返回 `bind_status=success` | OAuth 回调 | 授权页面自带超时处理 |
| 扫码添加鱼丸 | 用户扫码添加企微好友，企微回调通知服务端 | 服务端接口轮询（间隔 3s，最多 60s） | 提示"添加超时，请重试"，可手动点击"我已添加"触发二次检测 |
| 填写新人登记表 | 表单提交成功，API 返回 `submit_status=success` | 表单提交回调 | 提示"提交失败，请检查网络后重试" |

**交互逻辑**：
1. 用户点击操作按钮 → 按钮显示 loading spinner，不可重复点击
2. 执行操作（OAuth/扫码/填表）→ 系统通过回调/轮询检测完成
3. 检测到完成 → 按钮文案切换为"已完成"态，进度条 +1 并伴随 300ms 过渡动画
4. Toast 提示"XX 已完成"

**业务规则**：
- 以服务端状态为准，客户端乐观更新 + 服务端确认
- 同一用户在两个设备同时操作同一任务时，以先到达服务端的请求为准
- 用户切换页面或重新打开弹窗时，调用 API 刷新最新任务状态
- 已完成的任务不可回退到未完成状态
- 任务状态 API 需鉴权（Bearer Token），同一任务重复标记完成不产生副作用（幂等性）

**验收标准**：
- Given 用户完成"扫码添加鱼丸" When 服务端返回成功 Then 按钮在 300ms 内切换为"已添加"态，进度条同步更新
- Given 用户扫码添加鱼丸企微 When 企微回调到达服务端 Then 客户端在下次轮询时（< 3s）获取到完成状态并更新 UI
- Given 用户扫码后 60s 内未收到企微回调 When 轮询超时 Then 显示"添加超时，请重试"提示和"我已添加"手动确认按钮
- Given 用户在移动端完成某任务 When 用户切换到 PC 端打开弹窗 Then 该任务在 PC 端也显示为已完成
- Given 已完成的任务 When 用户尝试再次点击 Then 按钮不可点击，无任何响应
- Given 用户在功能上线前已完成某操作 When 首次触发新手任务 Then 该任务自动标记为已完成

**异常处理**：
- 任务完成操作失败：Toast 提示"操作失败，请检查网络后重试"，按钮恢复为可点击状态
- 企微回调超时（60s）：二维码弹窗下方显示"添加超时，请重试" + "我已添加"手动确认按钮
- 二维码加载失败：弹窗内占位区显示"加载失败，点击重试"

#### 3.2.3 首页新手任务弹窗

**Epic Hypothesis**：
If we 在首页自动弹出新手任务引导弹窗 for 有未完成新手任务的用户 then we will 实现首任务转化率达到 50% 以上

**功能描述**：
用户访问首页时，若存在未完成的新手任务，自动弹出引导弹窗。移动端弹窗引导跳转到任务列表页；PC 端弹窗内直接完成任务。

**交互逻辑**：
1. 用户访问首页 → 系统检查是否有未完成任务且本次会话未关闭过弹窗
2. 满足条件 → 延迟 600ms 后弹出弹窗
3. 移动端：弹窗展示标题 + "去完成"按钮 → 点击跳转任务列表页
4. PC 端：弹窗展示头部引导区 + 进度条 + 任务列表 → 在弹窗内完成任务
5. 点击关闭（×）→ 关闭弹窗 → PC 端显示悬浮提醒条
6. 所有任务完成 → 弹窗切换为庆祝界面 + 彩纸动画

**业务规则（触发与过期）**：

| 规则 | 说明 |
|------|------|
| 触发条件 | 用户访问首页时，若存在未完成的新手任务则触发弹窗（不限于首次登录） |
| 弹窗频率 | 每次会话仅弹出一次（关闭后本次会话不再自动弹出） |
| 过期规则 | 注册后 90 天内有效，超期后不再触发弹窗和提醒 |
| 老用户处理 | 功能上线前已注册的用户，若未完成对应操作，首次访问时同样触发 |
| 已完成操作的用户 | 系统根据已有数据自动标记已完成的任务 |

**PC 端"扫码添加鱼丸"交互**：
- 点击"去添加" → 弹窗内弹出二层弹窗展示鱼丸微信二维码
- 用户用手机微信扫码添加好友
- 系统通过企微回调检测添加成功 → 自动关闭二维码弹窗并更新任务状态
- 若 60s 内未收到回调 → 显示"添加超时，请重试"，用户可点击重试

**新人登记表功能整合**：
- 复用现有新人登记表页面/组件，不重复开发
- 用户通过任意入口完成填写后，新手任务中该项自动标记为已完成
- 若用户已通过原有入口填写过，进入新手任务时直接显示为"已填写"

**验收标准**：
- Given 用户有未完成的新手任务且本次会话未关闭过弹窗 When 进入首页 Then 600ms 延迟后弹出新手任务弹窗
- Given 用户在本次会话已关闭过弹窗 When 再次进入首页 Then 不弹出弹窗，PC 端显示悬浮提醒条
- Given 用户已完成全部新手任务 When 进入首页 Then 不弹出弹窗、不显示提醒条
- Given 用户注册已超过 90 天且未完成任务 When 进入首页 Then 不再触发新手任务弹窗

**异常处理**：
- 弹窗数据加载失败：不弹出弹窗，静默失败，不影响首页正常使用

#### 3.2.4 任务完成庆祝界面（P1）

**功能描述**：
所有任务完成后，页面/弹窗切换为庆祝界面，包含彩纸动画和"开始探索"按钮，给用户成就感和正向反馈。

**交互逻辑**：
1. 最后一个任务完成 → 状态更新完成后自动切换为庆祝界面
2. 显示彩纸动画 + "开始探索"按钮
3. 用户点击"开始探索" → 关闭弹窗/返回首页

**验收标准**：
- Given 用户刚完成最后一个任务 When 状态更新完成 Then 页面/弹窗切换为庆祝界面，显示彩纸动画和"开始探索"按钮
- Given 庆祝界面展示中 When 用户点击"开始探索" Then 关闭弹窗/返回首页

#### 3.2.5 悬浮提醒条（P1）

**功能描述**：
PC 端弹窗关闭后，在顶部导航栏下方显示悬浮提醒条，显示剩余任务数，引导用户重新打开弹窗继续完成任务。

**交互逻辑**：
1. 用户关闭新手任务弹窗且有未完成任务 → 顶部出现悬浮提醒条
2. 点击文字区域 → 重新打开新手任务弹窗
3. 点击关闭按钮 → 隐藏提醒条，关闭状态同步到服务端

**业务规则**：
- 弹窗关闭后显示，用户主动关闭后本次会话不再显示
- 关闭状态存储在服务端（跨设备生效），通过用户设置接口读写
- 全部任务完成后自动消失

**验收标准**：
- Given 用户关闭了新手任务弹窗但有未完成任务 When 弹窗关闭后 Then PC 端顶部出现悬浮提醒条，显示"你还有 X 个新手任务待完成"
- Given 悬浮提醒条展示中 When 用户点击文字区域 Then 重新打开新手任务弹窗
- Given 用户点击提醒条关闭按钮 When 关闭后 Then 本次会话不再显示，关闭状态同步到服务端

## 4. 非功能需求

### 4.1 性能要求
- 任务列表首屏渲染：< 1.5s（4G 网络，P90）
- 任务状态更新接口响应：< 500ms
- 任务状态最终一致性：< 3s

### 4.2 安全要求
- 任务状态 API 需鉴权（Bearer Token）
- 防重放：幂等性保证，同一任务重复标记完成不产生副作用
- 数据丢失率：< 0.01%

### 4.3 兼容性
- 移动端：iOS 14+, Android 10+, 微信内置浏览器 8.0+
- PC 端：Chrome 90+, Safari 15+, 最低分辨率 1280×720
- PC 端窗口宽度 < 768px 时弹窗自适应宽度

### 4.4 可扩展性
- 新增任务仅需后台配置（任务名、描述、跳转链接、完成检测方式），无需前端代码变更

## 5. 数据需求

### 5.1 核心数据实体

| 实体 | 关键字段 | 说明 |

## 6. 成功指标

### 主指标（只有一个）
**新手任务完成率** — 完成全部任务的用户数 / 触发新手任务的用户数
- 当前值：N/A（新功能）
- 目标值：>= 60%
- 衡量时间：上线后 30 天

### 辅助指标
- **单任务完成率**：每个任务各自的完成率 >= 70%
- **首任务转化率**：看到弹窗后点击"去完成"的比例 >= 50%
- **平均完成时长**：从首次看到弹窗到完成全部任务 < 10 分钟

### 护栏指标（不能变差）
- **首页跳出率**：弹窗不应导致首页跳出率上升
- **弹窗关闭率**：未做任何操作就关闭弹窗的比例 < 40%

## 7. 设计与交互要求

### 7.1 设计原则

- **移动端优先**：所有设计以 750px（2x）为基准
- **简洁温暖**：品牌主色生财绿 `--color-primary-6`，Nova 紧凑布局
- **渐进引导**：弹窗 → 任务列表 → 逐个完成，不一次性堆砌信息
- **正向反馈**：每完成一个任务都有视觉反馈（进度条 + Toast），全部完成有庆祝动画

### 7.2 关键页面/流程

> 详细 UI 设计见 `UI_Design_新手任务.md`（移动端）和 `UI_Design_新手任务_PC.md`（PC 端）
> 交互原型见 `prototype/index.html`（移动端）和 `prototype/新手任务-pc.html`（PC 端）

**业务流程**：

```
用户访问首页
 ├─ 检查：有未完成任务 + 注册 < 90 天 + 本次会话未关闭弹窗？
 │   ├─ 是 → 延迟 600ms 弹出弹窗
 │   │   ├─ 移动端：点击"去完成" → 任务列表页
 │   │   └─ PC 端：弹窗内展示任务列表，直接完成
 │   └─ 否 → 不弹出
 │
 ├─ 弹窗关闭（未完成全部任务）
 │   └─ PC 端：显示悬浮提醒条 → 点击可重开弹窗
 │
 ├─ 任务执行
 │   ├─ 绑定星球账号 → OAuth 授权页（仅移动端）
 │   ├─ 扫码添加鱼丸 → 二维码弹窗 → 企微回调检测
 │   └─ 填写新人登记表 → 复用现有表单 → 提交回调
 │
 └─ 全部完成 → 庆祝界面 → "开始探索" → 关闭弹窗/返回首页
```

## 9. 风险与依赖

### 9.1 本版本不做

| 功能/需求 | 不做的原因 |
|---------|---------|
| 任务完成奖励积分/勋章 | v1 先验证引导流程有效性，奖励系统另行规划 |
| 后台任务配置界面 | v1 任务写死在代码中，验证 PMF 后再做后台 |
| 任务排行榜/社交分享 | 非核心路径，v2 根据数据表现决定 |
| 推送通知提醒 | 避免过度打扰，v1 仅首页触发 |
| A/B 测试框架 | v2 规划 |

### 9.2 外部依赖
- 企业微信回调接口：扫码添加鱼丸任务的完成检测依赖企微回调
- 星球 OAuth 接口：绑定星球账号任务依赖星球 OAuth 授权
- 现有新人登记表功能：填写新人登记表任务直接复用，需确认接口可用

### 9.3 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|-----|-----|--------|
| 企微回调延迟或丢失 | 中 | 高 | 轮询超时后提供"我已添加"手动确认按钮；后台定时任务兜底校验 |
| 弹窗打扰导致用户跳出 | 低 | 高 | 弹窗延迟 600ms 展示；仅每次会话弹出一次；监控跳出率护栏指标 |
| 老用户历史数据不全 | 中 | 中 | 上线前批量校准：遍历已有企微好友关系和登记表提交记录，预填充完成状态 |
| 跨端状态不同步 | 低 | 中 | 以服务端为准，客户端每次打开弹窗/页面时重新拉取最新状态 |
