# PRD：深海圈客户管理

> **模块**：客户 > 客户管理 > 深海圈视图
> **版本**：v2.2
> **日期**：2026-04-01
> **原型**：`prototype/客户管理-交付扩展.html`（通过页签 Tab 切换「深海圈会员」）
> **状态**：草稿

---

## 1. 背景与目标

### 1.1 背景

CRM 系统区分三条业务线：**官网**、**深海圈**、**航海家**。其中官网和深海圈都有售前 + 交付两侧数据。

现有「客户管理」模块面向**官网**业务线，已实现销售侧 + 交付侧的统一视图（PRD_客户管理交付扩展.md）。深海圈作为独立的付费社群产品，其客户数据维度与官网有显著差异——核心关注**学习行为**、**社区活跃**、**社群活跃**和**运营标签**。

### 1.2 目标

在现有客户管理页面中，通过**面包屑页签（Tab）**切换「深海圈会员」视图：

- 页签：官网会员 | **深海圈会员** | 航海会员
- 点击「深海圈会员」Tab 后，搜索筛选项、表格列、详情抽屉全部切换为深海圈数据
- 点击其他 Tab 自动恢复对应视图

### 1.3 核心设计原则

- **同一页面，Tab 切换**：标题栏下方面包屑页签，不是独立页面
- **一人多圈子**：一个用户可同时属于多个深海圈，学习数据按圈子区分
- **详情对齐官网**：左右双栏布局、7 个 Tab、profile-table 样式，与官网保持一致
- **运营可操作**：自定义标签和运营备注支持编辑

## 5. 数据模型

### 5.1 一人多圈子

一个用户可同时属于多个深海圈（如"AI产品出海" + "B站好物"）。数据结构：

```json
{
  "userId": 100234,
  "circles": [
    {
      "name": "AI产品出海",
      "joinDate": "2025-06-15",
      "expireDate": "2026-06-14",
      "courseLearned": 8,
      "courseCompleted": 6,
      "courses": [...]
    },
    {
      "name": "B站好物",
      "joinDate": "2026-01-10",
      "expireDate": "2027-01-09",
      "courseLearned": 4,
      "courseCompleted": 2,
      "courses": [...]
    }
  ]
}
```

- 列表页「所属圈子」列显示主圈子名称
- 详情页/侧边栏概览中，圈子信息展示规则：每个圈子独立一组（所属圈子 + 会员状态 + 加入时间 + 到期时间）
- 学习行为按圈子分 section 展示
- 基础信息中不再重复显示所属圈子/加入时间/到期时间

#### 5.1.1 圈子展示规则

| 场景 | 展示逻辑 |
|------|---------|
| 所有圈子都是「正常」 | **全部展示**，每个圈子一组 |
| 部分「正常」部分「已过期」 | **只展示「正常」的圈子**，已过期不显示 |
| 只有一个「正常」 | **展示该正常圈子** |
| 所有圈子都「已过期」 | **只展示最晚到期的一个圈子** |

### 5.2 跨鱼丸数据可见性

> **会议决议 2026-04-01**：与之前按部门显示数据权限的方案不同，需单独处理。

当同一客户**同时添加了官网鱼丸和深海圈鱼丸**时，深海圈侧可查看该客户的**全部数据**（包括官网侧的客户信息、跟进记录、订单、行为动态等），不受部门数据权限限制。

| 场景 | 深海圈可见范围 |
|------|--------------|
| 客户仅添加深海圈鱼丸 | 仅深海圈相关数据 |
| 客户同时添加官网 + 深海圈鱼丸 | **全部数据**（官网 + 深海圈） |
| 客户仅添加官网鱼丸 | 不可见（深海圈侧无此客户） |

**影响范围**：
- 客户管理列表页：同时添加双鱼丸的客户，深海圈视图中可展示官网侧字段
- 详情抽屉：所有 Tab（客户档案/跟进记录/客户动态/积分记录/订单列表/变更记录/聊天记录）均展示全量数据
- 企微侧边栏（深海圈运营）：概览和各 Tab 展示全量数据

### 5.3 状态枚举

| 字段 | 枚举值 |
|------|--------|
| 会员状态 | 待付款 / 已付费 / 已确认 / 已完成 / 已驳回 / 已退款 / 已取消 |
| 用户活跃情况 | 连续一周不活跃 / 连续两周不活跃 / 超过一个月未活跃 |
| 用户成果标签 | 千刀户 / 万刀户 |

**会员状态说明**：

| 状态 | 说明 | 颜色方案 |
|------|------|---------|
| 待付款 | 已下单未付款 | 橙色（warning） |
| 已付费 | 已完成支付，待确认 | 蓝色（info） |
| 已确认 | 运营已确认，服务中 | 绿色（success） |
| 已完成 | 服务周期结束 | 灰色（neutral） |
| 已驳回 | 审核未通过 | 红色（danger） |
| 已退款 | 已退款处理 | 红色（danger） |
| 已取消 | 用户主动取消 | 灰色（neutral） |
