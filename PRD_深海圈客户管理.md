# PRD：深海圈客户管理

> **模块**：客户 > 客户管理 > 深海圈视图
> **版本**：v1.1
> **日期**：2026-03-30
> **原型**：`prototype/客户管理-交付扩展.html`（通过客户类型下拉选择「深海圈会员」切换）
> **状态**：草稿

## 1. 背景与目标

### 1.1 背景

CRM 系统区分三条业务线：**官网**、**深海圈**、**航海家**。其中官网和深海圈都有售前 + 交付两侧数据。

现有「客户管理」模块面向**官网**业务线，已实现销售侧 + 交付侧的统一视图（PRD_客户管理交付扩展.md）。深海圈作为独立的付费社群产品，其客户数据维度与官网有显著差异——核心关注**学习行为**、**社区活跃**、**社群活跃**和**运营标签**。

### 1.2 目标

在现有客户管理页面中，通过**「客户类型」下拉筛选**选择「深海圈会员」后，切换为深海圈视图模式：

- 搜索筛选项切换为深海圈专属（深海圈名称、身份、会员期数、活跃情况等）
- 表格列切换为深海圈字段（学习行为、社区活跃等）
- 详情抽屉展示深海圈专属区块（基础信息、学习行为、社区活跃、社群活跃、运营标签）

### 1.3 核心设计原则

- **同一页面，视图切换**：通过「客户类型」下拉筛选切换，不是独立页面
- **列表 + 详情**：列表展示关键字段，详情抽屉展示完整数据
- **运营可操作**：自定义标签和运营备注支持编辑

## 2. 数据模型

### 2.1 字段规格总览

共 5 大分类、30 个字段。

#### 2.1.1 基础信息（11 字段）

| # | 字段名 | 说明 | 数据来源 | 可编辑 |
|---|--------|------|----------|--------|
| 1 | 用户ID | 系统唯一标识 | CRM 后台 | 否 |
| 2 | 深海圈 | 所在深海圈名称 | CRM 后台 | 否 |
| 3 | 昵称 | 用户昵称 | CRM 后台 | 否 |
| 4 | 头像 | 用户头像 | CRM 后台 | 否 |
| 5 | 状态 | 正常 / 已退出 / 已移除 | CRM 后台 | 否 |
| 6 | 手机号 | 联系方式（脱敏展示） | 表单填写 | 否 |
| 7 | 微信号 | 微信触达标识 | 表单填写 | 否 |
| 8 | 身份 | 教练 / 志愿者 / 管理员 / 学员 | CRM 后台 | 否 |
| 9 | 加入时间 | 首次付费时间 | CRM 后台 | 否 |
| 10 | 到期时间 | 会员有效期截止日 | CRM 后台 | 否 |
| 11 | 会员期数 | 第几期加入（一期 / 二期 / 三期） | CRM 后台 | 否 |

#### 2.1.2 学习行为（6 字段）

| # | 字段名 | 说明 | 数据来源 |
|---|--------|------|----------|
| 1 | 已学课程数 | 进度 > 0 的课程卡片数，可展开查看具体课程及进度 | 官网 |
| 2 | 已完成课程数 | 进度 > 80% 的课程卡片数，可展开查看具体课程及进度 | 官网 |
| 3 | 直播观看数 | 已观看（进度 > 0）的直播回放数，可展开查看具体直播 | 官网 |
| 4 | 最近学习时间 | 最后一次课程或直播有进度变化的时间 | 官网 |
| 5 | 连续未学习天数 | 自动计算：今天 − 最近学习时间 | 计算字段 |
| 6 | 作业提交情况 | 累计提交作业数，含对应作品链接 | 官网 |

#### 2.1.3 社区活跃（6 字段）

| # | 字段名 | 说明 | 数据来源 |
|---|--------|------|----------|
| 1 | 累计发帖数 | 在社区发布的帖子总数 | 官网 |
| 2 | 累计评论数 | 在社区发布的评论总数 | 官网 |
| 3 | 累计获赞数 | 收到的点赞总数 | 官网 |
| 4 | 累计锚点数 | 收到的锚点打赏总数 | 官网 |
| 5 | 最近登录官网时间 | 最后一次访问深海圈官网的时间 | 官网 |
| 6 | 连续未登录官网天数 | 自动计算：今天 − 最近登录官网时间 | 计算字段 |

#### 2.1.4 社群活跃（2 字段）

| # | 字段名 | 说明 | 数据来源 |
|---|--------|------|----------|
| 1 | 微信群发言次数 | 统计周期内的群发言数 | 微信插件 |
| 2 | 最近群发言时间 | 最后一次在群里发言的时间 | 微信插件 |

#### 2.1.5 运营标签（5 字段）

| # | 字段名 | 说明 | 数据来源 | 可编辑 |
|---|--------|------|----------|--------|
| 1 | 用户期数 | 第一期 / 第二期 / 第三期 / 常态报名 | CRM 后台 | 否 |
| 2 | 用户活跃情况 | 连续一周不活跃 / 连续两周不活跃 / 超过一个月未活跃 | 学习行为计算 | 否 |
| 3 | 用户成果标签 | 千刀户 / 万刀户 | CRM 后台 | 否 |
| 4 | 自定义标签 | 运营手动打的标签（如企微渠道等） | 运营手动 | **是** |
| 5 | 运营备注 | 运营对该用户的备注信息 | 运营手动 | **是** |

### 2.2 状态枚举

| 字段 | 枚举值 |
|------|--------|
| 状态 | 正常 / 已退出 / 已移除 |
| 身份 | 教练 / 志愿者 / 管理员 / 学员 |
| 会员期数 | 一期 / 二期 / 三期 |

## 4. 详情抽屉

### 4.1 交互方式

| 属性 | 值 |
|------|-----|
| 触发 | 点击列表「详情」按钮 |
| 形式 | 右侧抽屉（Drawer） |
| 宽度 | 900px |
| 动画 | 打开：从右侧滑入 250ms ease-out；关闭：向右滑出 200ms ease-in |
| 遮罩 | 黑色 80%（`--color-overlay`），点击遮罩关闭 |
| 关闭 | 右上角 × 按钮 / 点击遮罩 |

### 4.2 布局结构

抽屉内部为**单栏布局**，从上到下依次为：

| 序号 | 区块 | 说明 |
|------|------|------|
| 1 | 用户概要 | 顶部固定区域：头像 + 昵称 + 状态标签 + 身份标签 |
| 2 | 基础信息 | 四栏表格展示 11 个字段 |
| 3 | 学习行为 | 六栏表格展示 6 个字段，课程列表可展开 |
| 4 | 社区活跃 | 四栏表格展示 6 个字段 |
| 5 | 社群活跃 | 四栏表格展示 2 个字段 |
| 6 | 运营标签 | 标签展示 + 备注编辑区 |

### 4.3 用户概要（顶部固定）

| 元素 | 说明 |
|------|------|
| 头像 | 56×56px，圆角 8px |
| 昵称 | 16px 加粗 |
| 深海圈名称 | 14px 灰色，昵称下方 |
| 状态标签 | 正常（绿 Tag）/ 已退出（灰 Tag）/ 已移除（红 Tag） |
| 身份标签 | 教练（橙 Tag）/ 志愿者（蓝 Tag）/ 管理员（紫 Tag）/ 学员（灰 Tag） |

### 4.5 学习行为区块

## 5. 非功能需求

| 项 | 要求 |
|----|------|
| 性能 | 列表页首屏加载 < 2s |
| 分页 | 默认 10 条/页，支持 20/50 条/页切换 |
| 导出 | 导出 Excel 包含全部基础信息 + 学习行为 + 运营标签字段 |
| 权限 | 深海圈客户管理的查看/编辑权限独立于官网客户管理 |
| 数据同步 | 学习行为和社区活跃数据从官网 API 定时同步（频率待定） |
