# PRD：企微侧边栏 · CRM 客户信息面板

> **版本**：v4.0 | **状态**：原型阶段 | **日期**：2026-03-06
> **负责人**：产品 — 明策 | **设计**：绘心

---

## 1. 文档概述

### 1.1 产品背景

销售/交付人员在企业微信中与客户聊天时，需要在聊天窗口右侧快速查看客户全貌信息（身份、状态、跟进历史、行为动态、订单记录），并能直接进行跟进记录等高频操作，减少在多系统间切换的成本。

### 1.2 产品目标

- 在企业微信聊天窗口侧边栏中，提供一站式客户信息查看与操作面板
- 让销售/交付人员在聊天场景下 3 秒内获取客户关键信息
- **销售组**通过三 Tab 导航（客户画像 / 客户动态 / 订单列表）覆盖客户转化全链路
- **交付组**通过三 Tab 导航（用户画像 / 行为动态 / 沟通记录）覆盖会员运营全链路
- 支持直接写跟进，无需跳转到 CRM 后台

### 1.3 核心指标

| 指标 | 目标值 |
|------|--------|
| 侧边栏打开后首屏信息加载时间 | < 1s |
| 单次跟进记录提交耗时 | < 30s |
| 销售人均日跟进记录数（使用侧边栏后） | 提升 40% |

### 1.4 范围

**In-Scope（本期）**：
- **销售组侧边栏**：客户画像 / 客户动态 / 订单列表（5 场景）
- **交付组侧边栏**：用户画像 / 行为动态 / 沟通记录（4 场景）
- 统计指标即筛选器（Option C）：点击统计数字直接筛选时间线
- 交付组两级筛选（方案 A）：一级按来源平台，二级按行为子项
- AI 画像总结（交付组：默认未生成，点击生成）
- 预警态处理（跟进过期、会员即将到期、用户沉默）
- 异常态处理（客户已删除好友，仅销售组）

**Out-of-Scope（后期）**：
- 待办工作台（独立 Tab 或独立页面）
- 三层标签体系（企微标签 / 行为标签 / 属性标签）
- CRM 客户列表页（PC 后台）
- 客户详情 Drawer（PC 后台）
- 聊天记录集成

## 2. 用户与场景

### 2.1 目标用户

| 角色 | 场景 | 核心诉求 | 使用侧边栏 |
|------|------|----------|-----------|
| 销售人员 | 与潜在/体验客户聊天时 | 快速了解客户等级、跟进状态，记录跟进内容 | **销售组**（CRM 客户管理） |
| 交付人员 | 与付费会员沟通时 | 查看会员活跃度、行为动态，推动会员成长 | **交付组**（CRM 会员运营） |
| 团队主管 | 查看团队成员的客户情况 | 了解客户状态、订单信息（只读） | 按角色切换 |

### 2.2 两侧边栏对比

同一用户在销售组和交付组有不同的视角：

| 维度 | 销售组（CRM 客户管理） | 交付组（CRM 会员运营） |
|------|----------------------|----------------------|
| 头部标题 | CRM 客户管理 | CRM 会员运营 |
| Tab 导航 | 客户画像 / 客户动态 / 订单列表 | 用户画像 / 行为动态 / 沟通记录 |
| 关注行为 | 官网社区行为（发帖/阅读/评论/点赞/投锚/收藏）+ 表单 | 全平台行为（官网社区 + 深海圈 + 航海 + 聚会） |
| 筛选方式 | Option C 一级：按行为类型筛选 | Option C 两级：一级按来源平台，二级按行为子项 |
| AI 画像 | 无 | 有（默认未生成，点击生成） |
| 高频参与项目 | 无 | 有（用户画像 Tab） |
| 活跃度 | 无 | 有（5 格网格） |
| 订单列表 | 有（独立 Tab） | 无 |
| 沟通记录 | 无 | 有（独立 Tab） |

### 2.3 用户故事

| 编号 | 角色 | 用户故事 | 优先级 |
|------|------|----------|--------|

| US-03 | 交付 | 在侧边栏中直观看到客户活跃度和沉默天数 | P0 |

| US-07 | 销售 | 切换到订单列表查看历史订单 | P0 |

## 3. 销售组侧边栏

### 3.1 整体结构

| 属性 | 值 |
|------|-----|
| 宽度 | 360px（企微侧边栏标准宽度） |
| 头部标题 | CRM 客户管理 |
| 头部操作 | 搜索、更多 |
| Tab 导航 | 客户画像（默认）/ 客户动态 / 订单列表 |
| 底部操作栏 | 仅客户画像 Tab 显示「写跟进」按钮 |

### 3.2 Tab 1 — 客户画像

内容区块按以下顺序排列：

#### 3.2.1 身份卡片

| 字段 | 数据来源 | 说明 |
|------|----------|------|
| 头像 | 企微同步 | 48x48 圆形，无头像时显示姓氏首字 + Teal 渐变背景 |
| 昵称 | 企微同步 | 16px/700，主文字色 |
| 星球编号 | 鱼丸系统 | 橙色胶囊标签，格式 `NO.28456`，非会员不显示 |
| 备注 | CRM 手动填写 | 13px/#757980，空态显示「点击添加备注」 |
| 手机号 | CRM 系统 | 12px/#757980，脱敏显示 `186****7890` |
| 入圈期数 | 鱼丸系统 | 12px/#757980，如「8体」 |

#### 3.2.2 状态标签栏

水平排列的状态标签，`padding: 0 16px 12px`，`gap: 6px`。

| 标签 | 颜色方案 | 说明 |
|------|---------|------|
| 客户状态 | 付费=绿，体验=橙，潜在=黄，流失=红 | CRM 系统 |
| 好友状态 | 灰色 | 企微同步 |
| 多人跟进 | 蓝色(#327CFB on #E8F4FF) | 多个销售同时跟进 |
| 分销 | 紫色(#7B61FF on #F0E8FF) | 分销客户 |

#### 3.2.3 体验营调研

展示「体验营课前调研」问卷数据。已填写问卷时显示，未填写则不显示。

| 字段 | 问卷题目 | 类型 |
|------|---------|------|
| 所在城市 | 第2题 | 文本 |
| 现在的工作 | 第3题 | 文本 |
| 工作年限 | 第4题 | 单选枚举 |
| 年收入 | 第5题 | 单选枚举 |
| 赚钱阶段 | 第6题 | 单选枚举 |

样式：浅主题色背景 #E9FFF8 + 主题色边框 #BCEDDD，圆角 8px。

#### 3.2.4 关键指标（3 项）

网格布局，3 项指标：

| 指标 | 数据来源 | 展示方式 | 预警规则 |
|------|----------|----------|----------|
| 客户等级 | CRM 系统 | ★星级（1-5）+ 字母等级 | 橙色实星，灰色空星 |
| 最近跟进 | CRM 系统 | 「N天前」 | > 7 天显示红色 |
| 体验营到期 | 鱼丸系统 | 日期 `YYYY-MM-DD` | 未参加显示「-」；剩余 ≤1 天红色 |

> v4.0 变更：从 4 项减为 3 项，移除「到期倒计时」，「会员到期」改为「体验营到期」。

#### 3.2.5 客户信息

| 字段 | 数据来源 |
|------|----------|
| 客户来源 | CRM 系统（如「合作导流-小红书」） |
| 鱼丸账号 | 鱼丸系统 |
| 专属销售 | CRM 系统 |
| 添加时间 | 企微同步 |

#### 3.2.6 最近跟进

时间线样式展示最近跟进记录。左侧圆点 8px，第一条主题色，后续灰色。

| 字段 | 说明 |
|------|------|
| 跟进方式 | 主题色文字（企微消息/电话/面谈等） |
| 时间 | MM-DD HH:mm |
| 跟进内容 | 13px/#242528 |
| 状态变更 | 如「状态变更：试用→付费」，橙色文字 |

#### 3.2.7 底部操作栏

固定底部，`height: 52px`。「写跟进」按钮（Primary 样式），点击展开跟进表单。

### 3.3 Tab 2 — 客户动态

#### 3.3.1 统计指标即筛选器（Option C）

6 项统计，3 列网格布局，每个格子可点击筛选时间线。

| 统计项 | data-filter | 说明 |
|--------|-------------|------|
| 发帖 | `post` | 官网社区发帖数 |
| 阅读 | `read` | 官网社区阅读数 |
| 评论 | `comment` | 官网社区评论数 |
| 点赞 | `like` | 官网社区点赞数 |
| 投锚 | `anchor` | 官网社区投锚数 |
| 收藏 | `collect` | 官网社区收藏数 |

**交互规则**：
- 点击统计数字 = 按该行为类型筛选时间线
- 单选切换，再次点击取消筛选（恢复全部）
- 激活态：主色边框 + 浅色背景 + 数值变色
- 无独立筛选栏，统计格充当筛选入口
- 筛选时隐藏无匹配项的日期分组

**数据范围**：仅统计官网社区行为，不含深海圈/航海/聚会/打卡（由交付组关注）。

#### 3.3.2 行为时间线

按日期分组，每组以日期标题开头。

**日期分组规则**：
- 今天 → 「今天 · YYYY-MM-DD」
- 昨天 → 「昨天 · YYYY-MM-DD」
- 前天及更早 → 仅显示日期「YYYY-MM-DD」
- 同日内按时间倒序

**各动态类型**：

| 类型 | data-activity | 圆点颜色 | 格式 |
|------|---------------|----------|------|
| 阅读内容 | `read` | 蓝 #327CFB | 阅读了 #{话题}「{标题}」+ 时长标签 |
| 收藏 | `collect` | 绿 #52C41A | 收藏了「{标题}」 |
| 表单填写 | `form` | 绿 #52C41A | 填写了表单「{名称}」 |
| 发表评论 | `comment` | 蓝 #327CFB | 在 #{话题} 评论了「{内容}」 |
| 投锚 | `anchor` | Teal #36A590 | 给 #{话题}「{标题}」投了锚 |
| 发表内容 | `post` | 蓝 #327CFB | 发表了 #{话题}「{标题}」 |

无底部操作栏。

### 3.4 Tab 3 — 订单列表

#### 3.4.1 搜索栏

输入框 30px 高，圆角 8px，占位符「🔍 搜索订单号/商品名称」，debounce 300ms 模糊搜索。

#### 3.4.2 状态筛选栏

| 筛选项 | 说明 |
|--------|------|
| 全部 N | 默认选中 |
| 待付款 N | 已下单未付款 |
| 已付款 N | 已完成支付 |
| 已完成 N | 已履约完成 |
| 已取消 N | 主动取消或超时关闭 |

选中态：主色文字 + 主色浅底。

#### 3.4.3 订单卡片列表

按下单时间倒序。每张卡片：订单编号 + 状态 Badge + 商品缩略图 + 商品名称 + 金额 + 下单时间。

无底部操作栏。

### 3.5 预警态（场景 4）

**触发条件**（满足任一）：

| 条件 | 阈值 |
|------|------|
| 最近跟进超期 | > 7 天（可配置） |
| 会员即将到期 | 剩余 ≤ 7 天（可配置） |
| 下次跟进已过期 | 下次跟进日期 < 当前日期 |

**视觉变化**：
- 关键指标区变为非对称布局（左列 1 项「会员到期时间」，右列 2 项「最近跟进」+「到期时间」），预警值红色
- 头像透明度 0.7
- 操作仍可用

### 3.6 异常态（场景 5）—— 客户已删除

客户已删除好友时：
- 头像灰色背景 + 50% 透明度
- 昵称颜色 #999
- 新增红色「客户已删除」标签
- 关键指标保留原有数据（4 项），值灰化
- 客户信息值灰化，新增「删除时间」字段（红色）
- 最近跟进显示空态
- 「写跟进」按钮禁用

### 3.7 写跟进表单

从底部展开，覆盖滚动内容区。

| 字段 | 类型 | 必填 |
|------|------|------|
| 跟进方式 | 单选 Chip | 是 |
| 跟进内容 | 文本域（1-500 字） | 是 |
| 状态变更 | 下拉选择 | 否 |
| 下次跟进 | 日期选择 | 否 |

## 4. 交付组侧边栏

### 4.1 整体结构

| 属性 | 值 |
|------|-----|
| 宽度 | 360px |
| 头部标题 | CRM 会员运营 |
| 头部操作 | 导出、设置 |
| Tab 导航 | 用户画像（默认）/ 行为动态 / 沟通记录 |
| 底部操作栏 | 用户画像 Tab 和沟通记录 Tab 显示「写跟进」按钮 |

### 4.2 Tab 1 — 用户画像

内容区块按以下顺序排列：

#### 4.2.1 身份卡片

与销售组相同结构：头像、昵称、星球编号、基础信息（城市/职业/行业）、备注。

#### 4.2.2 身份 & 阶段标签

| 标签类型 | 示例 | 说明 |
|---------|------|------|
| 身份标签 | 传术师、组局官 | 鱼丸系统身份 |
| 阶段标签 | 探索期 | 用户成长阶段 |

#### 4.2.3 AI 画像总结

**默认态**（未生成）：
- 显示 AI 图标 + 「画像总结」标签
- 下方显示「生成 AI 总结」按钮（主题色描边按钮）
- 点击后调用 AI 生成

**已生成态**：
- 显示总结文本（兴趣方向/优劣势/活跃度/风险）
- 底部「复制」「刷新」操作按钮
- 标签/行为/沟通变更后可重新生成

**预警态**：
- 背景变红色渐变
- 标签变「画像总结 · 流失预警」
- AI 增加流失风险分析和建议
- 直接展示（不需要手动生成）

#### 4.2.4 活跃度快照

5 列网格：

| 指标 | 数据来源 | 预警规则 |
|------|----------|----------|
| 连续沉默天数 | 官网访问记录 | > 7 天触发沉默预警 |
| 本月活跃天数 | 官网访问记录 | — |
| 发帖 | 鱼丸系统 | — |
| 航海 | 鱼丸系统 | — |
| 聚会 | 鱼丸系统 | — |

#### 4.2.5 高频参与项目

标签形式展示用户参与的航海/聚会项目，显示参与次数。

来源：鱼丸航海/聚会参与记录。不含退船记录。

> v4.0 变更：从行为动态 Tab 移至用户画像 Tab。

#### 4.2.6 标签

与销售组类似，支持「管理标签」入口。包含：
- 基础标签（普通在期圈友、续费期等）
- 兴趣标签（AI 自动打标：AI视频、自媒体、小红书等）

#### 4.2.7 最近跟进

时间线样式展示最近跟进记录（含 AI 自动总结标签）。

> v4.0 变更：「最近沟通」改名为「最近跟进」。

### 4.3 Tab 2 — 行为动态

#### 4.3.1 统计指标即筛选器（Option C + 两级方案 A）

4 项统计，4 列网格布局，按来源平台分类。每个格子可点击筛选时间线，并展开二级子筛选条。

**一级统计（按来源平台）**：

| 统计项 | data-filter | 数值含义 | 二级子项 |
|--------|-------------|----------|----------|
| 官网社区 | `community` | 该来源下所有行为合计 | 发帖/阅读/评论/点赞/投锚/收藏 |
| 深海圈 | `circle` | 该来源下所有行为合计 | 发帖/评论/点赞/投锚 |
| 航海 | `voyage` | 该来源下所有行为合计 | 参加/打卡/讨论 |
| 聚会 | `gathering` | 该来源下所有行为合计 | 报名/签到 |

**交互规则**：
- 点击一级统计 = 按来源平台筛选时间线
- 每个来源都展开二级子筛选条（含「全部」+ 各子项），可进一步按行为细分
- 单选切换，再次点击取消筛选（恢复全部）
- 二级子项显示子行为名称 + 数量
- 筛选时隐藏无匹配项的日期分组

**时间线 data-activity 规则**：
- 每个时间线条目携带空格分隔的多个标签，如 `"community-comment community"`
- 一级筛选匹配来源标签（如 `community`），二级筛选匹配具体行为标签（如 `community-comment`）

#### 4.3.2 行为时间线

与销售组相同的日期分组规则。收录行为（全量）：
- 社区：阅读/点赞/评论/提问/收藏/投锚/发帖/关注
- 航海：参与/打卡/日志/手册点击/手册评论
- 深海圈：加入/发帖/点赞/投锚/评论
- 聚会：参与/创建/填写反馈
- 私域：聊天/朋友圈互动
- 其他：填写表单

无底部操作栏。

### 4.4 Tab 3 — 沟通记录

#### 4.4.1 AI 聊天总结

梯度背景卡片，展示 AI 对近期沟通的分析：
- 友好度：★ 评分
- 续费意愿：文字描述
- 总结文本

#### 4.4.2 沟通时间轴

按时间倒序展示所有跟进记录，每条含 AI 自动总结标签、时间、内容摘要。

#### 4.4.3 备注栏

可编辑的备注文本区域。

底部显示「写跟进」按钮。

### 4.5 预警态（场景 4）

**触发条件**：
- 用户连续沉默 N 天（如 15 天）
- 活跃度显著下降
- 存在流失风险

**视觉变化**：
- 顶部红色预警横幅，描述预警原因
- 头像透明度 0.7
- AI 画像总结变红色风格，直接展示（无需点击生成），增加流失风险分析
- 活跃度指标关键数值红色高亮（连续沉默天数、本月活跃天数）
- 跟进记录显示较久远的时间（突出跟进不足）
- 操作仍可用

## 5. 共享规范

### 5.1 侧边栏头部

### 5.3 统计指标筛选器样式（Option C 通用）

| 属性 | 值 |
|------|-----|

## 7. 数据需求

### 7.1 数据来源

| 数据 | 来源系统 | 同步方式 |
|------|----------|----------|
| 头像、昵称、好友状态 | 企业微信 | 实时回调 |
| 星球编号、入圈期数、会员类型、到期时间 | 鱼丸系统 | API 查询 |
| 客户状态、等级、跟进记录 | CRM 系统 | 实时读写 |
| 客户行为动态 | 鱼丸平台 | 行为事件流 |
| 订单信息 | 商城订单系统 | API 查询 |
| 航海/聚会参与记录 | 鱼丸系统 | API 查询 |

### 7.2 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| 获取客户信息 | GET | 聚合身份、状态、指标、来源信息 |
| 提交跟进记录 | POST | 跟进方式 + 内容 + 状态变更 + 下次跟进 |
| 获取最近跟进 | GET | 最近 N 条跟进记录 |
| 获取客户动态 | GET | 按类型/来源筛选的行为事件列表（分页） |
| 获取订单列表 | GET | 按状态筛选的订单列表（支持搜索、分页） |
| 生成 AI 画像总结 | POST | 基于标签+行为+沟通生成画像分析 |
| 获取沟通记录 | GET | 聊天总结 + 跟进时间轴 |

## 8. 场景清单

### 8.1 销售组（5 场景）

| 场景 | 说明 | 特殊处理 |
|------|------|----------|
| 场景1：客户画像 | 体验客户，三 Tab 默认在客户画像 | 3 项关键指标，体验营调研，状态标签含「多人跟进」「分销」 |
| 场景2：客户动态 | 同一客户，切换到客户动态 Tab | Option C 统计筛选（6 项行为类型），无底部操作栏 |
| 场景3：订单列表 | 同一客户，切换到订单列表 Tab | 搜索+筛选+订单卡片，无底部操作栏 |
| 场景4：预警客户 | 即将到期 + 跟进过期 | 非对称指标布局，红色高亮，操作仍可用 |
| 场景5：异常态 | 客户已删除好友 | 文字灰化，操作禁用，显示删除时间 |

### 8.2 交付组（4 场景）

| 场景 | 说明 | 特殊处理 |
|------|------|----------|
| 场景1：用户画像 | 会员用户，三 Tab 默认在用户画像 | AI画像(默认未生成) → 活跃度 → 高频参与项目 → 标签 → 最近跟进 |
| 场景2：行为动态 | 同一用户，切换到行为动态 Tab | Option C 两级筛选（4 来源 × 子行为），无底部操作栏 |
| 场景3：沟通记录 | 同一用户，切换到沟通记录 Tab | AI聊天总结 + 沟通时间轴 + 备注 |
| 场景4：预警态 | 用户沉默/活跃下降 | 红色预警横幅，AI画像变红色直接展示，活跃度红色高亮 |
