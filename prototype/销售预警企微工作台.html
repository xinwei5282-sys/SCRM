销售预警企微工作台 PRD v1.1 · 2026-04-28

一线销售使用的企微侧任务处理页，对应 [PRD_销售预警企微工作台.md]
1. 页面定位
服务对象是一线销售，不是负责人或运营
目标是帮助销售在企微侧边栏里快速判断先处理谁、为什么处理、下一步做什么
本次预警口径聚焦客户未响应、付款停滞、激活断点、异议未闭环等业务风险
不承担团队经营分析、规则配置、复杂批量管理
2. 信息架构
Header：我的待处理预警（无摘要文案，避免与缺口概览重复）
缺口概览：一句话说明当前最危险的客户断点 + 期次切换
轻量统计：待处理 / 已超时 / 今日完成 / 执行完成率
今日优先：一句话摘要，告诉销售先处理哪类风险
分类筛选：全部 / 待处理 / 已超时（点击联动列表与标题）
预警卡片列表 + 已处理归档区
3. 顶部概览区
缺口概览
展示当前最核心风险，如“2 条客户未响应、1 条付款停滞、1 条异议待闭环”
文案只保留一个主判断，不展开复杂经营解释
轻量统计
辅助判断当前工作盘子大小，不替代任务列表
已超时、待处理需带风险语义
4. 筛选规则
全部：所有待处理卡片，标题「最紧急待处理」
高优先：客户未响应 / 高价值付款停滞 / 异议未闭环 / 已超时，标题「高优先待处理」
已超时：超过处理时限的任务，标题「已超时待处理」
切换筛选后列表标题同步更新，无关卡片隐藏
默认可见筛选项不超过 4 个
4a. 高优先判断条件
满足以下任一条件即标记为高优先
预警优先级为 P1（系统定义的最高级别预警）
客户 主动发起咨询超过 30 分钟未回复
高价值客户进入付款阶段后超过 24h 未推进
客户异议超过 6h 未处理或无下一步动作
客户价值标签为 高价值 且待处理超过 12h
展示规则
卡片左侧显示红色竖条，标题行出现「高优先」红色 tag
高优先卡片始终置顶，同优先级内按预警触发时间升序排列
4b. 已超时判断条件
满足以下任一条件即标记为已超时
预警触发后超过 处理时限仍未处理（各预警类型时限不同）
问卷提交未跟进：触发后 6h 未处理
付费未激活：触发后 24h 未处理
高价值付款停滞：触发后 72h 未推进
主动咨询未回复：触发后 1h 未回复
客户异议未闭环：触发后 24h 未闭环
任务状态为「执行中」超过预计执行时长 2 倍仍未完成
展示规则
时间信息变红，显示「⏰ 已超时 Xh」
已超时任务不自动消失，需人工处理或主管手动归档
同时满足高优先 + 已超时时，两个 tag 并列展示，卡片置顶
5. 卡片设计
默认态只展示：客户名、价值标签、预警类型、关键原因、主按钮、次按钮
关键原因优先说明“为什么现在要处理”
来源、时间、建议动作、策略建议通过“展开详情”呈现
每张卡片必须有明确主按钮，如“立即回复 / 确认进度 / 协助激活 / 处理异议”
6. 已处理归档
已完成项默认折叠进"已处理（N）"区，不与待处理争夺首屏焦点
展开后展示客户名、完成状态、处理结果摘要、后续动作入口
若任务虽已完成但仍有下一步，应保留继续跟进入口
已处理区不受筛选影响，始终在列表底部保持折叠
7. 验收标准
3 秒内能看懂当前风险缺口和第一优先处理方向
首屏可见 1-2 张待处理卡片，不被说明模块挤压
单卡默认信息量可扫读，不需要展开才能理解
点击高优先 / 已超时后，列表内容和标题同步更新，无关卡片隐藏
