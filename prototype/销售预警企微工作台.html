销售预警企微工作台 PRD v1.0 · 2026-04-22

一线销售使用的企微侧任务处理页，对应 [PRD_销售预警企微工作台.md]
1. 页面定位
服务对象是一线销售，不是负责人或运营
目标是帮助销售在企微侧边栏里快速判断先处理谁、为什么处理、下一步做什么
不承担团队经营分析、规则配置、复杂批量管理
2. 信息架构
Header：我的待处理预警（无摘要文案，避免与缺口概览重复）
缺口概览：一句话说明当前还差什么 + 期次切换
轻量统计：待处理 / 已超时 / 今日完成 / 执行完成率
今日优先：一句话摘要 + 查看策略入口
分类筛选：全部 / 高优先 / 已超时（点击联动列表与标题）
预警卡片列表 + 已处理归档区
3. 顶部概览区
缺口概览
展示当前最核心短板，如“还差 3 单问卷、1 单付款”
文案只保留一个主判断，不展开复杂解释
轻量统计
辅助判断当前工作盘子大小，不替代任务列表
已超时、待处理需带风险语义
4. 筛选规则
全部：所有待处理卡片，标题「最紧急待处理」
高优先：P1 或已超时或高价值黄金窗口内，标题「高优先待处理」
已超时：超过处理时限的任务，标题「已超时待处理」
切换筛选后列表标题同步更新，无关卡片隐藏
默认可见筛选项不超过 4 个
5. 卡片设计
默认态只展示：客户名、价值标签、预警类型、关键原因、主按钮、次按钮
关键原因优先说明“为什么现在要处理”
来源、时间、建议动作、策略建议通过“展开详情”呈现
每张卡片必须有明确主按钮，如“立即跟进 / 确认进度 / 去引导问卷”
6. 已处理归档
已完成项默认折叠进"已处理（N）"区，不与待处理争夺首屏焦点
展开后展示客户名、完成状态、处理结果摘要、后续动作入口
若任务虽已完成但仍有下一步，应保留继续跟进入口
已处理区不受筛选影响，始终在列表底部保持折叠
7. 验收标准
3 秒内能看懂当前缺口和第一优先处理方向
首屏可见 1-2 张待处理卡片，不被说明模块挤压
单卡默认信息量可扫读，不需要展开才能理解
点击高优先 / 已超时后，列表内容和标题同步更新，无关卡片隐藏
