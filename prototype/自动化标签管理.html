标签详情

自动化标签管理 PRD
当前视图：标签总览
1. 需求概述
让运营人员能查看标签体系 + 管理自动打标规则 + 编辑规则条件
标签来源：CDP 标签体系框架，4 大类 100 个标签（全局41+社区17+航海21+深海21）
规则来源：自动打标规则库，35 条规则
三个视图：标签总览（默认）/ 规则管理 / 规则编辑器（Drawer）
2. 标签总览
统计栏
4 个指标：总标签数 / 已启用规则数 / 今日更新标签数 / 覆盖率
分类 Tab
4 个分类（按产品线）：全局通用(41) / 官网社区(17) / 航海家(17) / 深海圈(21)
Tab 切换刷新卡片列表
标签卡片
Grid 布局每行 3 张
卡片信息：标签名 / 代码 / 取值范围 / 更新频率 / 关联规则数
点击卡片 → 弹出标签详情弹窗
3. 规则管理
表格列
规则名 / 标签分类 / 打标结果 / 条件摘要 / 频率 / 优先级 / 状态 / 操作
筛选
搜索框 + 分类下拉 + 状态下拉（启用/禁用/全部）
操作
编辑 → 打开规则编辑器 Drawer
复制 → 以副本方式打开
删除 → 确认后删除
状态 Toggle 开关 → 启用/禁用
4. 规则编辑器
基本信息
规则名称（必填）+ 规则描述（可选）
标签配置
分类 → 标签代码 → 标签值 三级联动
触发条件
5 种条件类型：属性 / 事件 / 标签 / 评分 / 人群包，组内 AND/OR，组间 且/或/且非
属性维度：会员等级 / 会员状态 / 客户等级 / 好友状态 / 续费状态 / 续费倾向分 / 到期倒计时 / 沉默天数 / 登录天数 / 入圈期数 / 下单次数 / 社群角色
自动移除条件（可选折叠）
同样的条件构建器，当条件满足时自动移除标签
执行配置
计算频率：实时 / 每日 / 每周
优先级：数字越小越优先（默认 10）
冷却期：天数，防止频繁打标
5. 校验规则
规则名称不能为空
必须选择标签分类和标签代码
触发条件至少有一个条件组
条件组内至少有一个条件
同一标签+标签值不能有重复的启用规则
6. 交互规则
编辑规则 → 预填已有数据
删除规则 → 二次确认弹窗
Toggle 状态 → Toast 提示
保存规则 → 校验 → Toast 成功/失败
预估匹配人数：基于条件模拟计算
标签详情弹窗：展示取值分布 + 关联规则列表
