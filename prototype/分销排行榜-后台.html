分销排行榜 · PRD v2.0
1. 需求概述
背景
KOC 通过分享链接邀请新用户下单，需实时排行榜展示分销战绩，形成竞争氛围
部分 KOC 拥有多个账号，需合并展示；部分渠道系统无法自动获取订单，需 Excel 导入补录
产品目标
激励 KOC 分销（转化率提升 20%）
排行榜实时更新，退款自动扣除
通用模板，支持多活动复用
2. H5 排行榜（微信内访问）
排行榜列表页
橙色渐变 Header + 活动标题 + 状态标签
我的排名卡片：登录后显示排名+头像+昵称+成交人数（可点击查看明细）
排行榜列表：排名 / 分销人(头像+昵称) / 星球编号 / 成交人数
前三名奖牌图标（🥇🥈🥉），当前用户行绿色高亮
排名算法
1. 按有效成交人数降序（有效 = 已付款 - 已退款）
2. 同分按首笔成交时间升序
3. KOC 多账号合并后统一计算
成交明细页
统计卡片：有效成交数 + 已退款数
成交人列表：头像 + 昵称 + 星球编号 + 订单状态（已付款绿/已退款红+删除线）
微信登录
静默授权（snsapi_base）+ 用户确认（snsapi_userinfo）
localStorage 存 token，有效期 7 天
3. 活动管理（后台 Tab 1）
活动列表
字段：活动名称 / 活动时间 / 状态（未开始/进行中/已结束）/ 排行榜链接（可复制）
操作：编辑 / 预览 / 订单（进入该活动订单管理子页面）
活动创建/编辑（居中弹窗）
活动名称 *（文本）
活动时间 *（日期范围）
活动 Banner 图（图片上传，建议 750×300）
活动规则 *（多行文本）
4. KOC 合并规则（后台 Tab 2）
业务说明
部分 KOC 使用多个微信号/手机号分销，通过合并规则将子账号数据归入主账号
规则列表
主账号（头像+昵称+编号）/ 子账号（tag 列表）/ 创建时间（精确到秒）/ 操作
规则编辑（居中弹窗）
主账号 + 子账号均通过搜索添加：星球编号 / 昵称 / 手机号 / 用户ID
选中显示为 tag（可 × 删除），删除按钮红色 hover
约束规则
一个账号只能出现在一条规则中 / 删除规则恢复独立 / 实时生效 / 跨活动全局配置
5. 订单管理（活动级子页面）
入口
从活动列表点击「订单」进入，顶部显示← 返回活动列表 + 活动名称 — 订单管理
筛选区（参考客户管理样式）
分销人搜索（昵称/星球编号/手机号）/ 成交人搜索 / 订单状态 / 订单来源 / 付款时间范围
操作按钮：查询 / 清空 / 导出 / Excel 导入
订单列表（列序对齐导入模板）
商户单号/订单号 — 双行显示（商户号上+订单号下），无则显示 —
订单状态 — 支付成功（绿）/ 已退款（红）
支付金额 — 加粗显示
订单来源 — 星橙(橙)/视频号(蓝)/官网(绿)/企微/支付宝/星球/Excel导入
用户信息 — 头像 + 昵称 + 星球编号（复用 customer-info 组件）
分享者 — 与用户信息同布局（头像 + 昵称 + 星球编号）
付款时间 / 退款时间（未退款显示 —）
订单来源（系统自动获取）
星橙订单 / 视频号订单 / 官网订单 / 企微订单 / 支付宝订单 / 星球订单
⚠️ 星球订单特殊说明 重要
星球订单无法获取收款单号，也无法关联退款单与原付款单的对应关系
因此星球订单的排行榜统计按人维度聚合：同一分享者邀请的同一用户，不论有多少笔订单，只计为 1 人
退款处理：如果该用户有任一退款记录，则从有效成交人数中扣除该人（而非按单扣除）
统计维度
排行榜统一按人统计（成交人数），不按订单笔数统计
同一用户通过不同渠道多次付款，仍只计为 1 人
6. Excel 批量导入
导入模板字段
A 商户订单号（去重）/ B 时间 / C 订单来源
D 成员编号（成交人星球编号）/ E 用户昵称 / F 支付金额
G 分享者编号（只填星球编号，系统自动匹配昵称头像）/ H 订单状态
导入流程
下载模板 → 填写数据 → 上传 Excel → 数据校验 → 预览结果 → 确认导入
校验规则
分享者编号须存在 / 商户订单号去重 / 付款时间须在活动范围内 / 状态仅接受 支付成功/已退款
7. 页面入口与导航
后台独立页面，侧边栏「营销 → 分销排行榜」
从其他后台页面（标签管理/人群包/自动化标签）侧边栏可跳转
两个 Tab：活动管理 / KOC 合并规则
订单管理为活动级子页面，从活动列表「订单」按钮进入
