## 1. 文档概述

### 1.1 产品背景

SCRM 系统中存在三种来源的客户标签：

1. **企微标签** — 企业微信侧边栏手动打标，同步到企微后台
2. **CRM 标签** — 系统内部业务标签，不同步企微，用于销售/服务内部分类
3. **自动化标签** — CDP 标签引擎自动计算，基于规则和模型生成（详见 `CDP_标签与评分框架.md`）

运营人员需要一个**统一的标签管理后台**，对三种标签进行查看、创建、编辑、删除和分组管理。

### 1.2 产品目标

- 提供统一的标签管理入口，覆盖企微/CRM/自动化三种标签来源
- 企微标签和 CRM 标签支持分组管理（两级树形结构）+ CRUD
- 自动化标签支持查看、搜索、启用/禁用，与自动化标签管理页面联动
- 为人群包条件引擎提供标签数据源（标签条件依赖此处的分组和取值）

### 1.3 核心指标

| 指标 | 目标值 |
|------|--------|
| 标签定位时间（从打开页面到找到目标标签） | < 15 秒 |
| 标签创建操作步骤 | ≤ 3 步 |
| 标签分组管理学习成本 | 无需培训 |

### 1.4 范围

**In-Scope（本期）**：
- 三 Tab 统一管理：企微标签 / CRM 标签 / 自动化标签
- 企微标签和 CRM 标签的分组树管理（两级）+ 标签 CRUD
- 自动化标签的查看、搜索、分类浏览、详情查看、启用/禁用
- 标签详情 Drawer（自动化标签）

**Out-of-Scope（后期）**：
- 标签批量导入/导出
- 标签合并/拆分
- 标签使用分析（哪些标签被人群包/MA 引用最多）
- 企微标签与 CRM 标签的自动映射

## 3. 企微标签

### 3.1 左侧 — 标签组树

**树形结构**：两级——一级为标签组，二级为子标签组。

**初始分组**：

| 一级组 | 二级组 |
|--------|--------|
| 转化组 | 意向客户、企微分组、用户分层 |
| 交付组 | 企微分组、售后跟进 |
| 运营组 | 活跃度、社群角色 |
| 来源组 | 渠道来源、活动来源 |
| 默认分组（不可删除） | 企微分组 |

**交互规则**：

| 操作 | 说明 |
|------|------|
| 点击「全部」 | 右侧显示所有标签 |
| 点击一级组 | 右侧显示该组下所有子组的标签 |
| 点击二级组 | 右侧仅显示该子组的标签 |
| Hover 一级组 | 显示操作：新建（子组）/ 编辑 / 删除 |
| Hover 二级组 | 显示操作：编辑 / 删除 |
| 搜索框 | 按分组名模糊过滤，高亮匹配项 |

**删除约束**：
- 默认分组不可删除
- 分组下存在标签时不可删除（提示「该分组下存在标签，不可删除」）
- 删除一级组时二次确认，会同时删除所有子组

### 3.2 右侧 — 标签列表

**表格列**：

| 列 | 宽度 | 说明 |
|----|------|------|
| ID | 60px | 标签 ID |
| 标签名称 | 自适应 | 标签显示名 |
| 人数 | 100px | 打上该标签的客户数，可点击 |
| 操作 | 120px | 修改 / 删除 |

**功能**：
- 搜索：按标签名模糊搜索
- 新建：弹窗输入标签名，自动归入当前选中的子组（未选中则归入默认分组）
- 修改：弹窗修改标签名
- 删除：二次确认后删除
- 分页：默认 5 条/页，支持切换 5/10/20/50 条/页，支持跳页

## 4. CRM 标签

### 4.1 左侧 — 标签组树

结构和交互与企微标签完全一致。

**初始分组**：

| 一级组 | 二级组 |
|--------|--------|
| 销售阶段 | 线索阶段、商机阶段、成交阶段 |
| 客户价值 | 价值分层、贡献度 |
| 服务管理 | 服务状态、满意度 |
| 运营分类 | 活动参与、内容偏好、社群活跃 |
| 默认分组（不可删除） | 未分组 |

### 4.2 右侧 — 标签列表

表格列、功能、交互均与企微标签一致。

### 4.3 与企微标签的区别

| 区别点 | 企微标签 | CRM 标签 |
|--------|---------|---------|
| 数据来源 | 企业微信 API 同步 | 系统内部创建 |
| 同步方向 | SCRM ↔ 企微双向同步 | 仅系统内部，不同步企微 |
| 用途 | 企微侧边栏可见，用于客户分层 | 销售/服务内部管理分类 |
| 标签上限 | 受企微 API 限制（每人最多 20 个标签组） | 无限制 |

| 指标 | 说明 |

| 标签覆盖率 | 至少有 1 个自动化标签的客户占比 |

### 5.2 左侧 — 分类树

**两级分类**（只读，不可编辑）：

| 一级分类 | 二级分类 | 标签数 |
|---------|---------|--------|
| 属性标签 | 基础身份 | 14 |
| | 交易属性 | 10 |
| | 计算属性 | 12 |
| | 偏好属性 | 8 |
| 行为标签 | 内容消费 | 6 |
| | 内容创作 | 4 |
| | 学习行为 | 12 |
| | 社交互动 | 7 |
| | 项目参与 | 8 |
| | 平台使用 | 9 |

- 点击一级分类：显示该大类下所有标签

**表格列**：

| 列 | 宽度 | 说明 |
|----|------|------|
| 标签名称 | 自适应 | 点击打开详情 Drawer |
| 标签代码 | 140px | 如 `lc_churn_risk`，等宽字体展示 |
| 取值范围 | 200px | 可选值列表（截断 2 行） |
| 更新频率 | 80px | 实时 / 每日 / 每周 / 每月 |
| 关联规则 | 80px | 关联的自动打标规则数 |
| 状态 | 70px | Toggle 开关（启用/禁用） |

**功能**：
- 搜索：按标签名或标签代码模糊搜索
- 新建标签：弹窗填写标签名/代码/分类/取值类型
- Toggle 状态：启用/禁用自动化标签
- 分页：默认 10 条/页

### 5.4 标签详情 Drawer

点击标签名称打开右侧 Drawer，包含三个区域：

**基本信息**：

| 字段 | 说明 |
|------|------|
| 标签名称 | 如「流失风险」 |
| 标签代码 | 如 `lc_churn_risk` |
| 所属分类 | 如「属性标签 > 计算属性」 |
| 数据来源 | 如「评分引擎」 |
| 更新频率 | 如「每日」 |
| 时间窗口 | 如「近 30 天」 |
| 说明 | 标签的业务含义描述 |

**取值逻辑**（表格）：

| 取值 | 判定条件 |
|------|---------|
| 高风险 | 续费倾向分 < 30 |
| 中风险 | 续费倾向分 30-59 |
| 低风险 | 续费倾向分 60-79 |
| 无风险 | 续费倾向分 ≥ 80 |

**关联规则**：
- 显示引用该标签的自动打标规则数
- 提供「查看详情 →」链接跳转到自动化标签管理页面

## 6. 数据结构

### 6.1 标签组（企微/CRM 共用）

```typescript
interface TagGroup {
  id: number;
  name: string;
  source: 'wecom' | 'crm';
  isDefault: boolean;            // 默认分组不可删除
  parentId?: number;             // 二级组的父 ID
  sortOrder: number;
  createdAt: string;
  updatedAt: string;
}
```

### 6.2 手动标签（企微/CRM）

```typescript
interface ManualTag {
  id: number;
  name: string;
  source: 'wecom' | 'crm';
  groupId: number;               // 所属二级组 ID
  customerCount: number;         // 标记的客户数
  wecomTagId?: string;           // 企微标签 ID（仅企微标签）
  createdAt: string;
  updatedAt: string;
}
```

### 6.3 自动化标签元数据

```typescript
interface AutoTagDefinition {
  code: string;                  // 标签代码，如 lc_churn_risk
  name: string;                  // 标签名称
  category: string;              // 二级分类 key
  description?: string;          // 说明
  source: string;                // 数据来源

  valueType: 'enum' | 'number' | 'text';
  values?: string;               // 取值范围描述
  logic?: {                      // 取值逻辑（enum 型）
    value: string;
    condition: string;
  }[];

  freq: string;                  // 更新频率
  timeWindow?: string;           // 时间窗口
  rules: number;                 // 关联规则数
  enabled: boolean;              // 启用状态

  createdAt: string;
  updatedAt: string;
}
```

## 7. 交互规范

### 7.1 Tab 切换

- 顶部三个 Tab 互斥，点击切换内容区
- 切换后保留当前 Tab 的筛选状态（搜索关键词、选中分组、分页位置）
- 默认显示第一个 Tab（企微标签）

### 7.2 分组树交互

| 操作 | 交互 |
|------|------|
| 选中分组 | 高亮当前选中项，右侧表格刷新 |
| Hover 显示操作 | 鼠标移入时显示操作链接（新建/编辑/删除） |
| 搜索 | 实时过滤，无匹配时隐藏分组 |
| 新建/编辑 | 居中弹窗，输入名称 |
| 删除 | confirm 二次确认 |

### 7.3 表格交互

| 操作 | 交互 |
|------|------|
| 搜索 | 输入框实时过滤，重置到第 1 页 |
| 新建标签 | 弹窗输入标签名 |
| 修改标签 | 弹窗修改标签名（预填当前值） |
| 删除标签 | confirm 二次确认 |
| 人数点击 | 企微/CRM 标签的人数可点击（预留跳转客户列表） |
| Toggle 状态 | 自动化标签的启用/禁用，Toast 提示 |
| 标签名点击 | 自动化标签名可点击，打开详情 Drawer |

### 7.4 详情 Drawer

- 从右侧滑入，宽度 480px
- 遮罩层点击或关闭按钮关闭
- 关闭动画：向右滑出 200ms

### 7.5 Toast 提示

所有操作成功/失败均使用居中 Toast 提示，1.2 秒后自动消失。

## 8. 校验规则

| 校验项 | 规则 |
|--------|------|
| 标签名称 | 必填，不可为空 |
| 标签名唯一性 | 同一来源（企微/CRM）同一分组内不可重名 |
| 分组名称 | 必填，不可为空 |
| 分组名唯一性 | 同一级别内不可重名 |
| 删除分组 | 分组下有标签时禁止删除 |
| 删除默认分组 | 默认分组不可删除 |

## 9. 与其他模块的关系

### 9.1 人群包（PRD_人群包.md）

人群包条件引擎的**标签条件**依赖标签管理的数据：
- 标签来源下拉：企微标签 / CRM 标签 / 自动化标签（来自三个 Tab）
- 标签组下拉：来自左侧分组树结构
- 标签值下拉：来自分组下的标签列表

### 9.2 自动化标签管理（自动化标签管理.html）

- 标签管理的「自动化标签」Tab 是**查看和浏览入口**
- 自动化标签管理页面是**规则配置入口**（规则编辑器、触发方式、变更日志）
- 两个页面通过侧边栏菜单互相链接

### 9.3 客户管理（客户管理-交付扩展.html）

- 客户列表的「标签」列显示该客户的标签
- 客户详情中的「打标签」操作调用企微/CRM 标签数据
- 标签筛选条件使用标签管理的分组和取值

### 9.4 企微侧边栏

- 企微标签在侧边栏中展示和操作
- 侧边栏打标时选择的标签来自企微标签分组

### A.2 全局通用 — 交易标签（txn_*）

| 标签代码 | 标签名 | 计算逻辑 | 取值规则 | 数据来源 | 更新频率 |
|---------|--------|---------|---------|---------|---------|
