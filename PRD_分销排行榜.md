# 分销排行榜 — 产品需求文档

> **版本**：v2.0 | **状态**：草稿 | **创建日期**：2026-04-01
> **产品线**：生财有术 SCRM — 营销工具
> **首期活动**：418 大促（2026-04-18）

---

## 1. 背景与目标

### 1.1 业务背景

生财有术定期举办大型促销活动（如 418、双 11），KOC（关键意见消费者）通过分享专属链接邀请新用户下单。为激励 KOC 分销，需要一个实时排行榜展示分销战绩，形成竞争氛围。

### 1.2 核心问题

- 部分 KOC 拥有多个账号（微信号/手机号不同），分销数据分散，无法体现真实排名
- 退款订单需实时扣除，避免刷单
- KOC 需要便捷查看自己的排名和成交明细
- 部分渠道系统无法自动获取订单，需通过 Excel 批量导入补录

### 1.3 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 激励 KOC 分销 | 活动期间分销转化率提升 20% |
| 透明化排名 | 排行榜数据实时更新，退款自动扣除 |
| 多账号合并 | KOC 多账号数据归一展示，排名准确 |
| 通用复用 | 模板化设计，5 分钟内可配置新活动 |

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|----------|
| KOC（分销人） | 通过分享链接邀请新用户下单的推广者 | 查看自己排名、了解成交明细 |
| 运营人员 | SCRM 后台管理员 | 配置活动、管理 KOC 合并规则、导入订单 |
| 普通用户 | 通过链接访问排行榜的非 KOC 用户 | 浏览排行榜（无明细权限） |

## 3. 用户故事

### KOC 视角

- **US-01**：作为 KOC，我想在微信中打开排行榜 H5，通过微信授权快速登录，查看我的实时排名
- **US-02**：作为 KOC，我想点击「我的排名」卡片的成交人数，查看我邀请成交的每个用户明细
- **US-03**：作为 KOC，我想看到退款订单被标记并从有效成交中扣除，确保排名公平
- **US-04**：作为拥有多个账号的 KOC，我希望所有账号的成交数据合并到同一排名，不被拆分

### 运营视角

- **US-05**：作为运营，我想在后台创建新活动并配置排行榜参数（活动名称、时间、Banner 图、活动规则）
- **US-06**：作为运营，我想通过星球编号/昵称/手机号/用户ID 搜索用户来维护 KOC 多账号合并规则
- **US-07**：作为运营，我想复用排行榜模板，为不同活动快速生成独立排行榜页面
- **US-08**：作为运营，我想通过 Excel 批量导入各渠道的成交订单，让排行榜数据完整

## 5. H5 前台 — 排行榜列表页

### 5.1 页面结构

```
┌──────────────────────────────┐
│  活动标题（如：418 分销排行榜）   │
│  活动状态 / 倒计时              │
├──────────────────────────────┤
│  🏆 我的排名                    │  ← 登录后显示
│  头像 昵称  #星球编号  第X名     │
│  成交 N 人 >                    │  ← 点击跳转明细
├──────────────────────────────┤
│  排行榜                         │
│  ┌────┬──────────┬──────┬────┐ │
│  │排名│分销人     │星球编号│成交│ │
│  ├────┼──────────┼──────┼────┤ │
│  │ 🥇 │头像 昵称A │#1234 │ 52 │ │
│  │ 🥈 │头像 昵称B │#5678 │ 41 │ │
│  │ 🥉 │头像 昵称C │#9012 │ 38 │ │
│  │ 4  │头像 昵称D │#3456 │ 35 │ │
│  └────┴──────────┴──────┴────┘ │
└──────────────────────────────┘
```

### 5.2 字段定义

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| 排名 | 按有效成交人数降序排列，相同成交数按首单时间升序 | 计算字段 |
| 分销人头像 | 微信头像 | 用户表 |
| 分销人昵称 | 微信昵称（KOC 合并后显示主账号昵称） | 用户表 |
| 星球编号 | 知识星球编号（主账号） | 用户表 |
| 成交人数 | 有效成交人数 = 总成交 - 已退款 | 订单表聚合 |

### 5.3 交互规则

| 规则 | 说明 |
|------|------|
| 未登录 | 显示排行榜列表，但"我的排名"区域显示"微信登录查看我的排名"按钮 |
| 已登录 | 顶部显示当前用户排名卡片；列表中自己的行高亮 |
| 非 KOC | 登录后"我的排名"显示"暂无分销记录" |
| 前三名 | 排名列显示奖牌图标（金🥇/银🥈/铜🥉） |
| 成交人数点击 | 已登录用户点击「我的排名」卡片的成交人数（带 > 箭头），跳转明细页 |
| 刷新 | 下拉刷新，数据实时更新 |
| 活动未开始 | 显示倒计时，列表显示"活动即将开始" |
| 活动已结束 | 显示"活动已结束"，数据冻结不再更新 |

### 5.4 排名算法

```
排名规则：
1. 按「有效成交人数」降序
2. 有效成交人数 = 已付款订单数 - 已退款订单数
3. 相同成交数时，按「首笔成交时间」升序（先达到的排前面）
4. KOC 多账号合并：所有子账号的成交数据归入主账号统计
5. 同一买家被多个子账号分别邀请：只计一次（归入主账号）
```

## 7. 微信登录

### 7.1 登录流程

```
用户访问 H5 → 检查登录态
  ├── 有 token → 直接加载数据
  └── 无 token → 微信网页授权
        ├── scope=snsapi_base（静默授权，仅获取 openid）
        └── 如需获取头像昵称 → scope=snsapi_userinfo（弹窗确认）
            → 获取 openid + 用户信息 → 换取业务 token → 加载数据
```

### 7.2 登录态管理

| 项目 | 说明 |
|------|------|
| 存储 | localStorage 存 token，有效期 7 天 |
| 刷新 | token 过期自动重新授权 |
| 退出 | 清除 localStorage，回到未登录状态 |

## 10. SCRM 后台 — KOC 合并规则

### 10.1 业务说明

部分 KOC 使用多个微信号/手机号分销，订单数据分散在不同账号。通过合并规则，将多个账号（子账号）的分销数据归入一个主账号，排行榜按主账号聚合展示。

### 10.2 规则列表

| 字段 | 说明 |
|------|------|
| 主账号 | KOC 的主展示账号（头像 + 昵称 + 星球编号） |
| 子账号 | 关联的子账号列表（tag 形式展示） |
| 创建时间 | 精确到时分秒（如 2026-04-01 10:23:15） |
| 操作 | 编辑 / 删除 |

### 10.3 规则编辑（居中弹窗）

主账号和子账号均通过搜索添加，支持以下搜索维度：
- 星球编号
- 昵称
- 手机号
- 用户 ID

选中后显示为 tag（头像 + 昵称 + 编号），可点击 × 删除。

### 10.4 合并规则约束

| 规则 | 说明 |
|------|------|
| 唯一性 | 一个账号只能出现在一条合并规则中（不能同时是主账号和子账号） |
| 级联删除 | 删除规则后，子账号数据恢复独立展示 |
| 实时生效 | 保存后排行榜立即按新规则重新聚合 |
| 跨活动 | 合并规则为全局配置，对所有活动生效 |

### 11.1 入口

从活动列表点击某活动的「订单」按钮，进入该活动的订单管理子页面。页面顶部显示「← 返回活动列表」按钮 + 活动名称。

| 筛选项 | 类型 | 说明 |
|--------|------|------|

| 成交人 | 文本搜索 | 昵称/星球编号/手机号 |
| 订单状态 | 下拉 | 支付成功 / 已退款 |
| 订单来源 | 下拉 | 星橙订单/视频号订单/官网订单/企微订单/支付宝订单/星球订单 |
| 付款时间 | 日期范围 | 开始 ~ 结束 |

操作按钮：查询 / 清空 / 导出 / Excel 导入

### 11.3 订单列表

列序和字段定义（对齐 Excel 导入模板）：

| 列 | 字段 | 说明 |
|----|------|------|
| 1 | 商户号/订单号 | 双行显示：商户号（上，深色）+ 订单号（下，浅色）；无订单号时显示 — |
| 2 | 订单状态 | 支付成功（绿色标签）/ 已退款（红色标签） |
| 3 | 支付金额 | 金额加粗显示 |
| 4 | 订单来源 | 星橙订单（橙）/ 视频号订单（蓝）/ 官网订单（绿）/ 企微订单 / 支付宝订单 / 星球订单 / Excel 导入 |
| 5 | 用户信息 | 头像 + 昵称（上）+ 星球编号（下），复用 customer-info 组件 |
| 6 | 分享者 | 与用户信息相同布局（头像 + 昵称 + 星球编号） |
| 7 | 付款时间 | 精确到时分 |
| 8 | 退款时间 | 已退款订单显示时间，未退款显示 — |

### 11.4 订单来源说明

**系统自动获取**

| 来源 | 说明 |
|------|------|
| 星橙订单 | 星橙平台自动同步 |
| 视频号订单 | 视频号自动同步 |
| 官网订单 | 官网自动同步 |
| 企微订单 | 企微支付自动同步 |
| 支付宝订单 | 支付宝自动同步 |
| 星球订单 | 知识星球自动同步 |

**人工补录**

| 来源 | 说明 |
|------|------|
| Excel 导入 | 运营通过 Excel 批量导入，导入后订单来源显示为具体渠道（如星橙订单） |

### 11.5 Excel 批量导入

**导入模板字段**

| 列 | 字段名 | 必填 | 说明 |
|----|--------|------|------|
| A | 商户订单号 | 是 | 第三方支付平台订单号，用于去重 |
| B | 时间 | 是 | 下单时间，格式：YYYY-MM-DDTHH:mm:ss |
| C | 订单来源 | 是 | 星橙订单 / 视频号订单 / 官网订单 / 企微订单 / 支付宝订单 / 星球订单 |
| D | 成员编号 | 是 | 成交人的星球编号，用于匹配用户 |
| E | 用户昵称 | 是 | 成交人昵称 |
| F | 支付金额 | 是 | 订单金额（元） |
| G | 分享者编号 | 是 | 分销人星球编号（如"11148"），系统根据编号自动匹配昵称和头像 |
| H | 订单状态 | 是 | 支付成功 / 已退款 |

**导入流程**

```
下载模板 → 填写数据 → 上传 Excel
  → 数据校验（格式、必填、星球编号匹配）
  → 预览校验结果（成功 N 条 / 失败 M 条 + 错误原因）
  → 确认导入
  → 导入完成，排行榜自动刷新
```

**校验规则**

| 规则 | 说明 |
|------|------|
| 分享者匹配 | 分享者编号（星球编号）必须在系统中存在，否则该行标记失败 |
| 重复检测 | 同一活动内，相同商户订单号视为重复，跳过 |
| 时间范围 | 付款时间须在活动时间范围内 |
| 状态校验 | 仅接受"支付成功"或"已退款" |
| 来源校验 | 必须为预定义的订单来源值 |
