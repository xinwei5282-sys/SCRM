# 分销排行榜 — 产品需求文档

> **版本**：v1.1 | **状态**：草稿 | **创建日期**：2026-04-01
> **产品线**：生财有术 SCRM — 营销工具
> **首期活动**：418 大促（2026-04-18）

---

## 1. 背景与目标

### 1.1 业务背景

生财有术定期举办大型促销活动（如 418、双 11），KOC（关键意见消费者）通过分享专属链接邀请新用户下单。为激励 KOC 分销，需要一个实时排行榜展示分销战绩，形成竞争氛围。

### 1.2 核心问题

- 部分 KOC 拥有多个账号（微信号/手机号不同），分销数据分散，无法体现真实排名
- 退款订单需实时扣除，避免刷单
- KOC 需要便捷查看自己的排名和成交明细
- 部分渠道（如线下推荐、私聊转账等）系统无法自动获取订单，需人工补录

### 1.3 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 激励 KOC 分销 | 活动期间分销转化率提升 20% |
| 透明化排名 | 排行榜数据实时更新，退款自动扣除 |
| 多账号合并 | KOC 多账号数据归一展示，排名准确 |
| 通用复用 | 模板化设计，5 分钟内可配置新活动 |

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|----------|
| KOC（分销人） | 通过分享链接邀请新用户下单的推广者 | 查看自己排名、了解成交明细 |
| 运营人员 | SCRM 后台管理员 | 配置活动、管理 KOC 多账号合并规则 |
| 普通用户 | 通过链接访问排行榜的非 KOC 用户 | 浏览排行榜（无明细权限） |

## 3. 用户故事

### KOC 视角

- **US-01**：作为 KOC，我想在微信中打开排行榜 H5，通过微信授权快速登录，查看我的实时排名
- **US-02**：作为 KOC，我想点击自己的排名行，查看我邀请成交的每个用户明细（头像、昵称、星球编号、订单状态）
- **US-03**：作为 KOC，我想看到退款订单被标记并从有效成交中扣除，确保排名公平
- **US-04**：作为拥有多个账号的 KOC，我希望所有账号的成交数据合并到同一排名，不被拆分

### 运营视角

- **US-05**：作为运营，我想在后台创建新活动并配置排行榜参数（活动名称、时间范围、关联商品）
- **US-06**：作为运营，我想维护 KOC 多账号合并规则（指定主账号，绑定子账号），合并后排行榜自动聚合
- **US-07**：作为运营，我想复用排行榜模板，为不同活动快速生成独立排行榜页面
- **US-08**：作为运营，我想通过 Excel 批量导入线下/私聊等渠道的成交订单，让排行榜数据完整
- **US-09**：作为运营，我想在后台手动添加单条订单记录（指定分销人和成交人），补录漏单

## 5. H5 前台 — 排行榜列表页

### 5.1 页面结构

```
┌──────────────────────────────┐
│  活动标题（如：418 分销排行榜）   │
│  活动状态 / 倒计时              │
├──────────────────────────────┤
│  🏆 我的排名                    │  ← 登录后显示
│  头像 昵称  #星球编号  第X名     │
│  成交 N 人                      │
├──────────────────────────────┤
│  排行榜                         │
│  ┌────┬──────────┬──────┬────┐ │
│  │排名│分销人     │星球编号│成交│ │
│  ├────┼──────────┼──────┼────┤ │
│  │ 🥇 │头像 昵称A │#1234 │ 52 │ │
│  │ 🥈 │头像 昵称B │#5678 │ 41 │ │
│  │ 🥉 │头像 昵称C │#9012 │ 38 │ │
│  │ 4  │头像 昵称D │#3456 │ 35 │ │
│  │ ...│          │      │    │ │
│  └────┴──────────┴──────┴────┘ │
└──────────────────────────────┘
```

### 5.2 字段定义

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| 排名 | 按有效成交人数降序排列，相同成交数按首单时间升序 | 计算字段 |
| 分销人头像 | 微信头像 | 用户表 |
| 分销人昵称 | 微信昵称（KOC 合并后显示主账号昵称） | 用户表 |
| 星球编号 | 知识星球编号（主账号） | 用户表 |
| 成交人数 | 有效成交人数 = 总成交 - 已退款 | 订单表聚合 |

### 5.3 交互规则

| 规则 | 说明 |
|------|------|
| 未登录 | 显示排行榜列表，但"我的排名"区域显示"微信登录查看我的排名"按钮 |
| 已登录 | 顶部显示当前用户排名卡片；列表中自己的行高亮 |
| 非 KOC | 登录后"我的排名"显示"暂无分销记录" |
| 前三名 | 排名列显示奖牌图标（金🥇/银🥈/铜🥉） |
| 点击行 | 仅已登录用户可点击自己的行进入明细页；其他行不可点击 |
| 刷新 | 下拉刷新，数据实时更新 |
| 活动未开始 | 显示倒计时，列表显示"活动即将开始" |
| 活动已结束 | 显示"活动已结束"，数据冻结不再更新 |

### 5.4 排名算法

```
排名规则：
1. 按「有效成交人数」降序
2. 有效成交人数 = 已付款订单数 - 已退款订单数
3. 相同成交数时，按「首笔成交时间」升序（先达到的排前面）
4. KOC 多账号合并：所有子账号的成交数据归入主账号统计
5. 同一买家被多个子账号分别邀请：只计一次（归入主账号）
```

## 7. 微信登录

### 7.1 登录流程

```
用户访问 H5 → 检查登录态
  ├── 有 token → 直接加载数据
  └── 无 token → 微信网页授权
        ├── scope=snsapi_base（静默授权，仅获取 openid）
        └── 如需获取头像昵称 → scope=snsapi_userinfo（弹窗确认）
            → 获取 openid + 用户信息 → 换取业务 token → 加载数据
```

### 7.2 登录态管理

| 项目 | 说明 |
|------|------|
| 存储 | localStorage 存 token，有效期 7 天 |
| 刷新 | token 过期自动重新授权 |
| 退出 | 清除 localStorage，回到未登录状态 |

## 9. SCRM 后台 — KOC 合并规则

### 9.1 业务说明

部分 KOC 使用多个微信号/手机号分销，订单数据分散在不同账号。通过合并规则，将多个账号（子账号）的分销数据归入一个主账号，排行榜按主账号聚合展示。

### 9.2 规则列表

| 字段 | 说明 |
|------|------|
| 主账号 | KOC 的主展示账号（头像 + 昵称 + 星球编号） |
| 子账号数 | 关联的子账号数量 |
| 创建时间 | 规则创建时间 |
| 操作 | 编辑 / 删除 |

### 9.3 规则编辑

```
┌──────────────────────────────┐
│  KOC 合并规则                  │
├──────────────────────────────┤
│  主账号（排行榜展示此账号信息）   │
│  [搜索选择用户 ▼]              │
│  → 头像 昵称A  #1234           │
├──────────────────────────────┤
│  子账号（这些账号的成交归入主账号）│
│  [搜索添加用户 ▼]              │
│  → 头像 昵称B  #5678  [×]     │
│  → 头像 昵称C  #9012  [×]     │
│  [+ 添加子账号]                │
├──────────────────────────────┤
│  [取消]           [保存]       │
└──────────────────────────────┘
```

### 9.4 合并规则约束

| 规则 | 说明 |
|------|------|
| 唯一性 | 一个账号只能出现在一条合并规则中（不能同时是主账号和子账号） |
| 级联删除 | 删除规则后，子账号数据恢复独立展示 |
| 实时生效 | 保存后排行榜立即按新规则重新聚合 |
| 跨活动 | 合并规则为全局配置，对所有活动生效 |

## 10. SCRM 后台 — 订单管理

### 10.1 业务说明

部分分销成交来自系统无法自动追踪的渠道（如线下推荐、微信私聊转账、其他平台代付等）。运营需要手动补录这些订单，确保排行榜数据完整。

### 10.2 数据来源标识

| 来源 | 标识 | 说明 |
|------|------|------|
| 系统自动 | `auto` | 通过邀请链接自动追踪的订单 |
| 手动录入 | `manual` | 运营在后台手动添加的单条订单 |
| Excel 导入 | `import` | 运营通过 Excel 批量导入的订单 |

### 10.3 订单列表

在活动详情内展示所有关联订单（系统 + 手动），支持按来源筛选。

| 字段 | 说明 |
|------|------|
| 分销人 | 头像 + 昵称 + 星球编号 |
| 成交人 | 头像 + 昵称 + 星球编号 |
| 订单状态 | 已付款 / 已退款 |
| 数据来源 | 系统自动 / 手动录入 / Excel 导入 |
| 成交时间 | 下单时间 |
| 操作 | 手动订单可编辑/删除，系统订单只读 |

### 10.4 手动添加订单

```
┌──────────────────────────────┐
│  手动添加订单                   │
├──────────────────────────────┤
│  活动          418 大促（自动带入）│
│  分销人 *      [搜索选择用户 ▼]  │
│  成交人 *      [搜索选择用户 ▼]  │
│  星球编号       [输入成交人星球编号]│
│  订单状态       ○ 已付款  ○ 已退款│
│  成交时间 *     [日期选择器]      │
│  备注          [备注说明]        │
├──────────────────────────────┤
│  [取消]              [保存]     │
└──────────────────────────────┘
```

### 10.5 Excel 批量导入

**导入模板字段**

| 列 | 字段名 | 必填 | 说明 |
|----|--------|------|------|
| A | 商户订单号 | 是 | 第三方支付平台订单号，用于去重 |
| B | 时间 | 是 | 下单时间，格式：YYYY-MM-DDTHH:mm:ss |
| C | 订单来源 | 是 | 支付宝 / 企微 / 星球 等 |
| D | 成员编号 | 是 | 成交人的星球编号，用于匹配用户 |
| E | 用户昵称 | 是 | 成交人昵称 |
| F | 支付金额 | 是 | 订单金额（元） |
| G | 分享者(编号) | 是 | 分销人昵称 + 星球编号，如"更绿(11148)" |
| H | 订单状态 | 是 | 支付成功 / 已退款 |

**导入流程**

```
下载模板 → 填写数据 → 上传 Excel
  → 数据校验（格式、必填、星球编号匹配）
  → 预览校验结果（成功 N 条 / 失败 M 条 + 错误原因）
  → 确认导入
  → 导入完成，排行榜自动刷新
```

**校验规则**

| 规则 | 说明 |
|------|------|
| 分销人匹配 | 星球编号必须在系统中存在，否则该行标记失败 |
| 重复检测 | 同一活动内，相同分销人+成交人+成交时间视为重复，跳过 |
| 时间范围 | 成交时间须在活动时间范围内 |
| 状态校验 | 仅接受"已付款"或"已退款" |
