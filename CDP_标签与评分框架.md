## 1. 客户生命周期模型

### 1.1 生命周期阶段定义

知识付费/付费社群的客户生命周期与电商、SaaS 存在本质差异：核心产品是**年度会员制**（而非一次性购买），用户价值通过**内容消费深度 + 社群参与度 + 续费行为**三维衡量。

```
┌─────────────────────────────────────────────────────────────────────┐
│                     生财有术客户生命周期                              │
│                                                                     │
│  访客 → 潜在客户 → 体验客户 → 付费会员 → 活跃会员 → 核心会员        │
│   │        │          │          │          │          │             │
│   │        │          │          │          │     ┌────┘             │
│   │        │          │          │          │     ↓                  │
│   │        │          │          │     沉默会员 → 流失风险           │
│   │        │          │          │          │          │             │
│   │        │          │          │          └──→ 到期未续 → 流失     │
│   │        │          │          │                                   │
│   │        │          │          └──→ 续费会员 ──→ (回到活跃会员)     │
│   │        │          │                                              │
│   └── 流失(从未转化)  └── 体验流失                                    │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 阶段详细定义与转换触发器

| 阶段 | 代码 | 定义 | 进入触发器 | 退出触发器 | 典型停留时长 |
|------|------|------|-----------|-----------|------------|
| **访客** | `visitor` | 通过任意渠道触达但未建立联系 | 点击广告/内容/直播间 | 添加企微好友 or 注册鱼丸账号 | 0-7 天 |
| **潜在客户** | `prospect` | 已建立企微好友关系但未进入体验流程 | 添加销售企微 | 报名体验营 or 直接购买会员 | 1-30 天 |
| **体验客户** | `trial` | 正在参加体验营（如 7 天体验营） | 报名体验营 + 领取体验卡 | 体验营到期 or 购买正式会员 | 7-14 天 |
| **付费会员** | `paid_member` | 已购买正式会员（在期） | 支付会员订单成功 | 会员到期 + 未续费超 30 天 | 365 天（年费制） |
| **活跃会员** | `active_member` | 付费会员且近期有持续互动 | 近 30 天内 >= 3 次有效行为 | 连续 30 天无有效行为 | — |
| **核心会员** | `core_member` | 高价值活跃会员（传术师/航海家/高频内容生产者） | 满足核心会员评分阈值 | 评分降低至阈值以下 | — |
| **沉默会员** | `silent_member` | 付费会员但近期无互动 | 连续 30 天无有效行为 | 恢复活跃 or 到期 | — |
| **流失风险** | `churn_risk` | 会员即将到期且参与度低 | 到期前 60 天 + 沉默会员 | 续费 or 正式流失 | 30-90 天 |
| **续费会员** | `renewed` | 已续费的会员（标记，非独立阶段） | 完成续费支付 | 进入新一轮活跃/沉默判断 | — |
| **到期未续** | `expired` | 会员到期后未续费 | 会员到期 + 未续费 | 续费 or 超 180 天标记流失 | 0-180 天 |
| **流失** | `churned` | 到期超 180 天未续费 or 删除好友 | 到期 180 天 + 未续费 | 重新购买 | — |

### 1.3 有效行为定义

以下行为计入「活跃」判断：

| 行为 | 权重 | 说明 |
|------|------|------|
| 登录 | 1 | 任意端登录 |
| 阅读 | 1 | 阅读帖子（停留 >= 30 秒） |
| 发帖 | 5 | 在社区发表内容 |
| 评论 | 3 | 在帖子下评论 |
| 点赞 | 1 | 对帖子/评论点赞 |
| 投锚 | 2 | 对帖子投锚 |
| 收藏 | 1 | 收藏帖子 |
| 参加航海 | 5 | 报名并参加航海活动 |
| 观看直播 | 2 | 观看直播 >= 5 分钟 |
| 直播评论 | 3 | 在直播间发言 |
| 搜索 | 1 | 使用搜索功能 |

> **有效行为阈值**：近 30 天内行为权重总和 >= 5 即视为「活跃」。

### 1.4 阶段转换规则引擎配置

```typescript
interface LifecycleTransitionRule {
  from: LifecycleStage;
  to: LifecycleStage;
  conditions: {
    type: 'attribute' | 'event' | 'computed';
    field: string;
    operator: string;
    value: any;
    timeWindow?: TimeWindow;
  }[];
  logic: 'and' | 'or';
  priority: number;          // 多规则匹配时优先级
  autoExecute: boolean;      // 是否自动执行
  cooldown?: number;         // 冷却期（天），防止频繁切换
}
```

**核心转换规则示例**：

| 转换 | 条件 | 自动执行 | 冷却期 |
|------|------|---------|--------|
| 潜在 → 体验 | 报名体验营 AND 领取体验卡 | 是 | 无 |
| 体验 → 付费 | 支付会员订单成功 | 是 | 无 |
| 付费 → 活跃 | 近 30 天行为权重 >= 5 | 是（每日计算） | 7 天 |
| 活跃 → 沉默 | 连续 30 天行为权重 = 0 | 是（每日计算） | 7 天 |
| 沉默 → 流失风险 | 会员到期前 <= 60 天 AND 近 30 天行为权重 < 5 | 是 | 无 |
| 付费/活跃 → 到期未续 | 会员到期日 < 今天 AND 未续费 | 是 | 无 |
| 到期未续 → 流失 | 到期超 180 天 AND 未续费 | 是 | 无 |
| 任意 → 潜在 | 重新购买 or 重新添加好友 | 否（人工确认） | 30 天 |

## 2. RFM 模型（知识付费适配版）

### 2.1 RFM 指标定义

知识付费场景下，传统 RFM 的 Monetary 需要重新定义——因为年费制产品的金额差异较小，**内容贡献度**和**社群参与度**比消费金额更能反映用户价值。

因此，本框架采用 **RF(M+E)** 模型（Recency + Frequency + Monetary + Engagement 加权）：

| 维度 | 传统电商定义 | 知识付费适配定义 | 数据来源 |
|------|------------|----------------|---------|
| **R — Recency** | 最近一次购买距今天数 | 最近一次**有效互动**距今天数 | 行为事件表 |
| **F — Frequency** | 一段时间内购买次数 | 近 90 天内**有效行为天数**（去重登录日） | 行为事件表 |
| **M — Monetary** | 一段时间内消费总额 | **综合价值 = 累计消费金额 + 内容贡献积分** | 订单表 + 鱼丸积分 |

> **为什么用「有效行为天数」而非「行为次数」**：知识付费用户的一天多次刷帖与一天深度阅读 1 篇的价值接近。使用「天数」消除高频低质行为的噪音。

### 2.2 指标详细计算

**R — Recency（最近互动天数）**

```
R = 今天 - 最近一次有效行为日期
```

有效行为包括：登录、阅读(>30s)、发帖、评论、点赞、投锚、收藏、参加航海、观看直播(>5min)。

**F — Frequency（互动频率）**

```
F = 近 90 天内有有效行为的天数（去重日期计数）
```

**M — Monetary（综合价值）**

```
M = 累计消费金额 × 0.6 + 内容贡献积分 × 0.4

内容贡献积分 = 发帖数 × 10 + 评论数 × 3 + 投锚数 × 5 + 被收藏数 × 2 + 航海参与数 × 15
```

> 权重可配置。消费金额占 60% 是因为知识付费的核心商业模式仍是付费，但内容贡献是社群健康度的关键。

### 2.3 五级评分体系

每个维度按百分位数分为 5 档（1-5 分）：

| 分值 | R（天数，越小越好） | F（天数，越大越好） | M（综合值，越大越好） |
|------|-------------------|-------------------|---------------------|
| **5** | 0-3 天 | >= 60 天 | Top 5%（>= P95） |
| **4** | 4-7 天 | 30-59 天 | P75-P94 |
| **3** | 8-14 天 | 15-29 天 | P50-P74 |
| **2** | 15-30 天 | 5-14 天 | P25-P49 |
| **1** | > 30 天 | 0-4 天 | Bottom 25%（< P25） |

> 注意：R 分值是反向的——越近越高分。

### 2.4 RFM 用户分群

基于 R/F/M 三维评分，定义 8 个核心用户群：

| 用户群 | R | F | M | 代码 | 描述 | 运营策略 |
|--------|---|---|---|------|------|---------|
| **超级用户** | 高 | 高 | 高 | `rfm_champion` | 近期活跃、高频互动、高价值 | VIP 专属服务，邀请成为传术师/航海家 |
| **忠实用户** | 中高 | 高 | 中高 | `rfm_loyal` | 稳定活跃，价值较高 | 维持互动，引导内容创作 |
| **潜力用户** | 高 | 低中 | 低中 | `rfm_potential` | 最近活跃但历史频次/价值不高 | 引导深度参与，推荐航海和话题 |
| **新用户** | 高 | 低 | 低 | `rfm_new` | 刚付费/刚活跃 | 新手引导，推送入门内容 |
| **需关注** | 中 | 中 | 中 | `rfm_attention` | 各维度中等，有上升或下滑的可能 | 定期触达，关注趋势变化 |
| **即将沉默** | 低中 | 中高 | 中高 | `rfm_about_to_sleep` | 历史活跃但近期减少 | 唤回干预，个性化推荐 |
| **沉默用户** | 低 | 低中 | 低中 | `rfm_hibernating` | 很久没有互动 | 强唤回（短信/企微消息），限时优惠 |
| **流失用户** | 低 | 低 | 低 | `rfm_lost` | 所有维度都低 | 最后挽回尝试，或放弃投入 |

### 2.5 RFM 分群阈值矩阵

| 分群 | R 分值范围 | F 分值范围 | M 分值范围 |
|------|----------|----------|----------|
| 超级用户 | 4-5 | 4-5 | 4-5 |
| 忠实用户 | 3-5 | 4-5 | 3-5 |
| 潜力用户 | 4-5 | 1-3 | 1-3 |
| 新用户 | 4-5 | 1 | 1-2 |
| 需关注 | 2-3 | 2-3 | 2-3 |
| 即将沉默 | 1-2 | 3-5 | 3-5 |
| 沉默用户 | 1-2 | 1-3 | 1-3 |
| 流失用户 | 1 | 1 | 1-2 |

### 2.6 计算频率

| 计算类型 | 频率 | 触发方式 |
|---------|------|---------|
| 全量计算 | 每日凌晨 3:00 | 定时任务 |
| 增量计算 | 实时 | 关键事件触发（支付、发帖、登录） |
| 阈值校准 | 每月 1 日 | 基于全量数据重新计算百分位 |

## 3. 自动化标签分类体系

### 3.1 标签分类架构

> **v2.0 重构**：从按数据类型分类（身份/行为/交易/生命周期/偏好/社交 6 类）改为**「全局底座 + 产品线专属」**架构。
> 运营人员按产品线思考，而非按数据来源思考。参考小鹅通、得到、混沌学园等行业标杆实践。

```
标签体系（v2.0 — 产品线架构）
│
├── 全局通用（Global）                    ← 跨产品线共享，含身份/交易/生命周期
│   ├── 身份标签 id_*                     ← 系统属性，基本不变
│   ├── 交易标签 txn_*                    ← 订单/支付数据
│   └── 生命周期标签 lc_*                 ← 模型计算 + 跨平台偏好
│
├── 官网社区（Community）comm_*           ← 内容消费 + 创作 + 社区影响力
│   适用：鱼丸社区帖子阅读、发帖、评论、点赞、投锚、搜索、直播
│   说明：官网是主平台，包含航海项目（付费课程）入口
│
├── 航海家（Navigator）nav_*              ← 课程学习 + 俱乐部权益 + 课程偏好
│   适用：航海训练营报名、课程学习、作业提交、打卡、实操产出
│   说明：类似高级官网会员，拥有专属俱乐部和特定权益（航海是付费课程体系）
│
└── 深海圈（DeepSea）deep_*              ← 高阶项目参与 + 成果产出 + 角色协作
    适用：高阶项目参与、导师/学员角色、跨项目协作、校友网络
    说明：深海圈是高阶版航海，面向进阶用户的项目制深度学习
```

**为什么不按数据类型分类？**

| 对比维度 | 按数据类型（v1.0） | 按产品线（v2.0） |
|---------|-------------------|------------------|
| 组织逻辑 | 身份/行为/交易/生命周期/偏好/社交 | 全局通用/官网社区/航海家/深海圈 |
| 运营视角 | 需跨 6 个分类找标签 | 按产品线直觉查找 |
| 跨产品线分析 | 天然支持但标签含义模糊 | 全局层统一，产品线层精确 |
| 典型场景 | "阅读深度"不区分在哪个产品线 | `comm_read_depth` vs `deep_discussion_depth` 语义清晰 |

### 3.2 全局通用标签（Global Tags）

跨产品线共享的基础标签，含身份属性、交易数据、生命周期状态。

#### 3.2.1 身份标签（id_*）

基于用户基础属性自动生成，通常不需要复杂规则。

| 标签代码 | 标签名 | 取值示例 | 数据来源 | 更新频率 |
|---------|--------|---------|---------|---------|
| `id_member_level` | 会员等级 | 体验会员 / 正式会员 / 老会员(续费>=2次) | 鱼丸系统 | 实时 |
| `id_member_period` | 入圈期数 | 第 8 期、第 9 期 | 鱼丸系统 | 订单触发 |
| `id_customer_level` | 客户等级 | S / A / B / C / D | CRM 系统 | 手动/规则 |
| `id_role` | 社群角色 | 普通圈友 / 传术师 / 航海家 / 管理员 | 鱼丸系统 | 系统同步 |
| `id_source_channel` | 来源渠道 | 视频号-坤哥直播 / 公众号-文末导流 / 合作导流-小红书 | CRM 系统 | 首次添加 |
| `id_source_type` | 来源类型 | 自然流量 / 付费投放 / KOC推荐 / 老带新 | CRM 系统 | 首次添加 |
| `id_referrer` | 推荐人 | 用户ID/昵称 | CRM 系统 | 首次添加 |
| `id_sales_owner` | 专属销售 | 员工名 | CRM 系统 | 分配时 |
| `id_service_owner` | 专属服务 | 员工名 | CRM 系统 | 分配时 |
| `id_wechat_status` | 好友状态 | 好友 / 客户删除 / 随身删除 / 未添加 | 企微同步 | 实时 |
| `id_trial_batch` | 体验营期数 | 9体139 / 9体137 | CRM 系统 | 报名时 |
| `id_region` | 所在城市 | 北京、上海、深圳 | 体验营调研 | 填写时 |
| `id_occupation` | 职业类型 | 互联网 / 金融 / 自由职业 / 实体经营 | 体验营调研 | 填写时 |
| `id_income_tier` | 收入层级 | <10w / 10-30w / 30-50w / 50-100w / >100w | 体验营调研 | 填写时 |

#### 3.2.2 交易标签（txn_*）

基于订单和支付数据生成。

| 标签代码 | 标签名 | 规则 | 取值 | 更新触发 |
|---------|--------|------|------|---------|
| `txn_total_spent` | 累计消费金额 | 历史所有订单支付金额之和 | 高消费(>=10000) / 中(3000-9999) / 低(<3000) | 支付成功 |
| `txn_purchase_count` | 购买次数 | 历史支付成功订单数 | 多次(>=5) / 几次(2-4) / 首次(1) | 支付成功 |
| `txn_latest_product` | 最近购买产品 | 最近一次购买的产品名称 | 年费会员 / 航海门票 / 深海圈项目 / 周边 | 支付成功 |
| `txn_renewal_count` | 续费次数 | 会员续费次数（不含首次） | 0 / 1 / 2 / 3+ | 续费成功 |
| `txn_renewal_status` | 续费状态 | 当前续费状态 | 首年 / 已续费 / 待续费(到期前90天) / 已过期 | 每日计算 |
| `txn_clv_tier` | CLV 层级 | 客户终身价值层级 | 钻石(Top5%) / 金(Top20%) / 银(Top50%) / 铜(Bottom50%) | 月度计算 |
| `txn_avg_order_value` | 平均客单价 | 历史平均每单金额 | 具体金额 | 支付成功 |
| `txn_refund_flag` | 退款标记 | 是否有退款记录 | 有退款 / 无退款 | 退款事件 |
| `txn_member_duration` | 会员时长 | 累计会员天数（含所有续费期） | 新(<=180天) / 中(181-730天) / 老(>730天) | 每日计算 |
| `txn_expire_countdown` | 到期倒计时 | 距会员到期天数 | 具体天数（负数=已过期天数） | 每日计算 |

#### 3.2.3 生命周期标签（lc_*）

基于生命周期模型和评分模型自动计算，含跨平台偏好。

| 标签代码 | 标签名 | 规则 | 取值 | 更新频率 |
|---------|--------|------|------|---------|
| `lc_stage` | 生命周期阶段 | 见 1.2 阶段定义 | visitor / prospect / trial / paid_member / active_member / core_member / silent_member / churn_risk / expired / churned | 每日 |
| `lc_health` | 健康度等级 | 见 4.1 健康分模型 | 健康(>=80) / 良好(60-79) / 一般(40-59) / 风险(20-39) / 危险(<20) | 每日 |

## 4. 评分模型体系

### 4.1 客户健康分（Customer Health Score）

综合衡量客户当前状态，预测续费/流失概率。

**评分范围**：0-100 分

**计算公式**：

```
Health Score =
    活跃度得分 × 0.30
  + 内容消费得分 × 0.20
  + 社群参与得分 × 0.15
  + 交易健康得分 × 0.20
  + 服务体验得分 × 0.15
```

**各维度评分细则**：

| 维度 | 权重 | 满分 100 分的评估要素 |
|------|------|---------------------|
| **活跃度** | 30% | 近 30 天登录天数(40%) + 近 7 天是否登录(30%) + 活跃趋势(30%) |
| **内容消费** | 20% | 近 30 天阅读篇数(50%) + 平均阅读时长(30%) + 内容类型多样性(20%) |
| **社群参与** | 15% | 近 30 天发帖(30%) + 评论(30%) + 航海参与(20%) + 直播观看(20%) |
| **交易健康** | 20% | 会员状态(30%) + 续费次数(20%) + 累计消费(20%) + 到期倒计时(30%) |
| **服务体验** | 15% | 跟进响应率(40%) + 最近跟进时效(30%) + 投诉/退款记录(30%) |

**健康度等级划分**：

| 等级 | 分值范围 | 颜色 | 含义 | 运营策略 |
|------|---------|------|------|---------|
| 健康 | 80-100 | 绿色 #52C41A | 状态良好，续费概率高 | 维持现有服务，引导升级 |
| 良好 | 60-79 | 浅绿 #76DB45 | 基本正常，有提升空间 | 提升参与度，个性化推荐 |
| 一般 | 40-59 | 黄色 #FA9A1D | 需要关注 | 主动触达，推送感兴趣内容 |
| 风险 | 20-39 | 橙色 #FF7D00 | 有流失风险 | 1v1 关怀，续费优惠 |
| 危险 | 0-19 | 红色 #F54A45 | 高流失风险 | 紧急干预，管理层介入 |

### 4.2 续费倾向分（Renewal Propensity Score）

预测会员续费概率，仅对**在期会员**计算。

**评分范围**：0-100 分

**计算公式**：

```
Renewal Score =
    使用活跃度 × 0.25
  + 内容参与深度 × 0.20
  + 历史续费行为 × 0.20
  + 社群融入度 × 0.15
  + 到期紧迫度 × 0.10
  + 消费投入度 × 0.10
```

**核心指标**：

| 指标 | 权重 | 满分标准 | 零分标准 |
|------|------|---------|---------|
| 近 30 天活跃天数 | 25% | >= 20 天 | 0 天 |
| 近 90 天深度阅读篇数(>3min) | 20% | >= 50 篇 | 0 篇 |
| 历史续费次数 | 20% | >= 3 次 | 0 次（首年） |
| 航海/活动参与次数（近 180 天） | 15% | >= 3 次 | 0 次 |
| 距到期天数 | 10% | > 180 天（充足） | 已过期 |
| 累计消费金额 | 10% | >= 10000 | < 1000 |

**续费风险等级**：

| 等级 | 续费倾向分 | 标签 | 预计续费概率 | 干预优先级 |
|------|----------|------|------------|----------|
| 无风险 | 80-100 | `低流失风险` | > 85% | P3 - 常规维护 |
| 低风险 | 60-79 | `低流失风险` | 60-85% | P2 - 关注 |
| 中风险 | 30-59 | `中流失风险` | 30-60% | P1 - 主动干预 |
| 高风险 | 0-29 | `高流失风险` | < 30% | P0 - 紧急挽回 |

### 4.3 参与度评分（Engagement Score）

衡量用户在社群中的整体参与深度。

**评分范围**：0-100 分

**计算公式**：

```
Engagement Score =
    内容消费分 × 0.25
  + 内容创作分 × 0.25
  + 互动行为分 × 0.20
  + 项目参与分 × 0.15
  + 直播参与分 × 0.15
```

**各维度明细**：

| 维度 | 满分条件（100 分） | 计算逻辑 |
|------|-------------------|---------|
| 内容消费 | 近 30 天阅读 >= 60 篇 | 线性映射，上限封顶 |
| 内容创作 | 近 30 天发帖 >= 10 篇 | 发帖(×10) + 精华帖(×20) |
| 互动行为 | 近 30 天评论+点赞+投锚 >= 50 次 | 评论(×3) + 投锚(×2) + 点赞(×1) + 收藏(×1) |
| 项目参与 | 近 90 天参加航海 >= 2 次 | 每次航海 50 分，上限 100 |
| 直播参与 | 近 30 天观看 >= 5 场且评论 >= 10 次 | 观看(×10) + 评论(×5) |

**参与度层级**：

| 层级 | 分值 | 代码 | 用户画像 | 占比预期 |
|------|------|------|---------|---------|
| L1 - 核心 | 80-100 | `eng_core` | 内容创作+深度互动+航海参与 | 5-8% |
| L2 - 活跃 | 60-79 | `eng_active` | 经常阅读+定期互动+偶尔创作 | 15-20% |
| L3 - 参与 | 40-59 | `eng_engaged` | 定期阅读+少量互动 | 25-30% |
| L4 - 浏览 | 20-39 | `eng_browser` | 偶尔阅读+很少互动 | 25-30% |
| L5 - 沉默 | 0-19 | `eng_silent` | 很少或不访问 | 15-25% |

### 4.4 客户价值分（Customer Lifetime Value Score）

预测客户全生命周期价值。

**评分范围**：0-100 分

**计算公式**：

```
CLV Score =
    已实现价值 × 0.40
  + 预测未来价值 × 0.35
  + 社交传播价值 × 0.25

已实现价值 = 累计消费金额标准化(0-100)
预测未来价值 = 续费倾向分 × 预期续费年数 × 年费金额标准化
社交传播价值 = (推荐付费人数 × 客单价 + 内容影响力分) 标准化
```

**CLV 层级**：

| 层级 | 分值 | 标签 | 说明 | 服务级别 |
|------|------|------|------|---------|
| 钻石 | 85-100 | `clv_diamond` | Top 5% 客户 | 专属 1v1 服务 |
| 铂金 | 70-84 | `clv_platinum` | Top 5-15% | 优先服务 |
| 金 | 50-69 | `clv_gold` | Top 15-35% | 标准增值服务 |
| 银 | 30-49 | `clv_silver` | Top 35-65% | 标准服务 |
| 铜 | 0-29 | `clv_bronze` | Bottom 35% | 基础服务 |

### 4.5 评分仪表板指标

供管理层和运营团队监控的关键汇总指标：

| 指标 | 计算方式 | 预警阈值 |
|------|---------|---------|
| 平均健康分 | 全体付费会员健康分均值 | < 60 分预警 |
| 健康分布 | 各等级人数占比 | 「风险+危险」占比 > 20% 预警 |
| 续费倾向分布 | 各风险等级人数及占比 | 「高风险」占比 > 15% 预警 |
| 即将到期且高风险 | 30 天内到期 + 续费倾向分 < 30 | 绝对人数 > 50 预警 |
| 参与度趋势 | 本月 vs 上月 L1+L2 占比变化 | 下降 > 5% 预警 |
| 新客 30 天留存 | 注册后 30 天仍活跃的比例 | < 50% 预警 |

### 5.9 规则优先级与冲突处理

当多条规则命中同一标签类别时：

| 冲突类型 | 处理策略 | 示例 |
|---------|---------|------|
| 同标签多值 | 取最高优先级规则的值 | 沉默 30 天 vs 60 天 → 取 60 天 |
| 互斥标签 | 后触发的覆盖先触发的 | 活跃回归 覆盖 沉默用户 |
| 补充标签 | 可叠加 | 航海达人 + 精华作者 同时生效 |
| 跨产品线标签 | 互不影响 | `comm_creator_tier` 和 `nav_completion_rate` 独立计算 |

### 6.2 数据层（Data Layer）

负责采集和汇聚所有客户相关数据。

#### 6.2.1 数据源清单

| 数据源 | 数据类型 | 采集方式 | 更新频率 | 核心字段 |
|--------|---------|---------|---------|---------|
| **企微** | 好友关系、聊天事件 | API 同步 | 准实时 | 好友状态、添加时间、标签、所属员工 |
| **鱼丸系统** | 用户属性、社区行为 | API + 事件推送 | 准实时 | 星球编号、入圈期数、发帖/阅读/评论等行为 |
| **订单系统** | 订单、支付、退款 | API + Webhook | 实时 | 订单号、商品、金额、支付状态、时间 |
| **直播系统** | 直播观看、评论 | 事件推送 | 准实时 | 直播间、观看时长、评论内容 |
| **CRM 跟进** | 跟进记录、状态变更 | 系统内部 | 实时 | 跟进方式、内容、时间、操作人 |
| **体验营** | 报名、完成、调研 | API | 事件触发 | 期数、报名时间、到期时间、调研问卷 |
| **行为埋点** | 页面访问、点击、搜索 | SDK 上报 | 实时 | 事件名、属性、时间戳、设备信息 |

#### 6.2.3 ID 映射表（Identity Resolution）

一个客户可能有多个 ID，需要建立映射关系：

| ID 类型 | 来源 | 唯一性 | 示例 |

#### 6.3.2 画像更新策略

| 更新类型 | 触发条件 | 更新内容 | 延迟要求 |
|---------|---------|---------|---------|
| 实时更新 | 关键事件（支付、好友变更） | 会员状态、好友状态 | < 5 秒 |
| 准实时更新 | 行为事件（阅读、发帖） | 行为统计计数器 | < 1 分钟 |
| 批量更新 | 每日定时任务 | 评分、标签、生命周期阶段 | 每日凌晨 |
| 按需更新 | 人群包查询、画像查看 | 衍生指标 | < 5 秒 |

### 6.4 计算层（Computation Layer）

负责运行各种模型和规则，生成标签和评分。

#### 6.4.1 标签引擎

```
输入：统一客户画像 + 自动打标规则库
输出：标签集合更新

执行流程：
1. 读取所有 enabled 的自动打标规则
2. 按 priority 排序
3. 对每个客户，逐规则评估条件
4. 命中规则 → 执行 action（add/remove/update）
5. 处理冲突（同类标签取高优先级）
6. 写入标签变更日志
7. 更新画像中的标签集合
```

#### 6.4.2 评分引擎

```
输入：统一客户画像（行为统计 + 交易统计 + 标签）
输出：四维评分更新

执行流程：
1. 拉取客户最新行为和交易数据
2. 按各模型公式计算原始分
3. 标准化到 0-100
4. 生成评分等级标签
5. 写入评分历史（用于趋势分析）
6. 更新画像中的评分集合
```

#### 6.4.3 生命周期引擎

```
输入：统一客户画像 + 生命周期转换规则
输出：生命周期阶段更新

执行流程：
1. 读取当前生命周期阶段
2. 评估所有从当前阶段出发的转换规则
3. 检查冷却期限制
4. 执行阶段转换
5. 记录转换日志（from, to, trigger, timestamp）
6. 触发阶段变更事件（供 MA 系统消费）
```

#### 6.4.4 规则引擎

```
输入：事件流 + 规则库
输出：触发动作（打标、通知、任务创建）

支持的规则类型：
- 条件规则：满足条件 → 执行动作
- 定时规则：按 cron 表达式定期执行
- 事件触发规则：特定事件发生时执行
- 组合规则：多条件 AND/OR 组合
```

### 6.5 应用层（Application Layer）

面向运营人员的产品功能。

#### 6.5.1 人群圈选（已有 — PRD_人群包）

与现有人群包规则构建器对接，支持：
- 基于标签条件圈选（`身份标签.会员等级 = S`）
- 基于评分条件圈选（`健康分 < 40`）
- 基于行为事件圈选（`做过 发帖 >= 5次 近30天`）
- 基于人群包交叉圈选

#### 6.5.2 营销自动化（MA）

基于标签和评分变化触发自动化流程：

| 触发场景 | 触发条件 | 自动化动作 |
|---------|---------|-----------|
| 续费提醒 | 到期前 30 天 + 续费倾向 < 50 | 企微消息 + 销售任务 |
| 沉默唤回 | 沉默 30 天 + 健康分下降 | 个性化内容推送 |
| 新人引导 | 新付费 + 未完成新手任务 | 系列引导消息（D1/D3/D7） |
| 体验转化 | 体验营过半 + 高意向标签 | 销售跟进任务 |
| 流失挽回 | 到期未续 + 历史高参与度 | 限时优惠 + 1v1 联系 |

#### 6.5.3 客户画像（已有 — 侧边栏）

在企微侧边栏中展示关键标签和评分：
- 生命周期阶段标签
- 健康分仪表盘（颜色编码）
- 续费风险标识
- 偏好话题和学习阶段
- RFM 分群标识

#### 6.5.4 数据看板（Analytics）

运营管理级别的数据分析：
- 客户总量及各生命周期阶段分布
- 健康分分布趋势（按周/月）
- 续费漏斗（窗口期 → 提醒 → 意向 → 续费）
- RFM 分群迁移矩阵（本月 vs 上月）
- 标签覆盖率（各标签类别的覆盖率和分布）
- 预警看板（高风险用户列表和趋势）
