# PRD：MA 人群包

> **版本**：v4.0 | **状态**：草稿 | **日期**：2026-03-25
> **负责人**：产品 — 明策 | **设计**：绘心
> **参考**：Twilio Segment、Adobe AEP、神策、GrowingIO、火山引擎 CDP

---

## 1. 文档概述

### 1.1 产品背景

营销自动化（MA）需要精准圈选目标用户群体。v2.0 仅支持「属性条件 + 2 层嵌套（组 > 条件）」，无法满足以下常见场景：

- **行为筛选**：找出「近 30 天内发帖 >= 5 次」的活跃用户
- **组合运算**：对多个人群包做交集、并集、差集
- **灵活逻辑**：组内 AND/OR 可切换，覆盖复杂业务规则

v3.0 对齐行业 CDP 标准（神策/Segment/Adobe AEP），引入事件条件和人群包条件，同时保持 2 层嵌套的简洁性。

### 1.2 产品目标

- 支持属性条件、事件条件、人群包条件、标签条件、评分条件五种条件类型
- 支持条件组间 OR、组内 AND/OR 可切换
- 支持满足区 + 排除区，实现交集、并集、差集运算
- 提供可视化规则构建器，运营人员无需技术背景即可配置

### 1.3 核心指标

| 指标 | 目标值 |
|------|--------|
| 规则配置完成时间（中等复杂度） | < 3 分钟 |
| 人群包预估计算响应时间 | < 5 秒 |
| 规则构建器上手学习成本 | 无需培训，参考同类产品交互 |

### 1.4 范围

**In-Scope（本期）**：
- 规则构建器 UI（满足区 + 排除区 + 条件组 + AND/OR 切换）
- 五种条件类型：属性条件、事件条件、人群包条件、标签条件、评分条件
- 时间窗口体系（7 种类型）
- 预估人数功能
- 人群包 CRUD + 手动/自动刷新

**Out-of-Scope（后期）**：
- 实时人群包（流式计算）
- 人群包洞察分析（属性分布、行为画像）
- 人群包对比功能
- Lookalike 扩展

### 1.5 与 v2.0 的关键变更

| 项 | v2.0 | v3.0 | v4.0 |
|-----|------|------|------|
| 条件类型 | 仅属性条件 | 属性 + 事件 + 人群包 | **+ 标签 + 评分（共 5 种）** |
| 组合运算 | 无 | 支持交集、并集、排除 | 同 v3.0 |
| 嵌套深度 | 2 层（组 > 条件） | 2 层 + 组内 AND/OR 可切换 | 同 v3.0 |
| 时间窗口 | 仅日期类维度 | 标准化 7 种时间窗口类型 | 同 v3.0 |
| CDP 对齐 | — | — | **与自动化标签共享统一条件引擎** |

## 2. 规则逻辑模型

### 2.1 整体结构

参考神策/Segment 的「条件组」模型，支持组内 AND/OR 切换：

```
人群包定义
├── 满足以下条件的用户（include）
│   ├── 条件组 1 [AND/OR 可切换]
│   │   ├── 条件 A
│   │   ├── 条件 B
│   │   └── 条件 C
│   │          ↕ 组间关系: OR（固定）
│   └── 条件组 2 [AND/OR 可切换]
│       ├── 条件 D
│       └── 条件 E
│
└── 排除以下条件的用户（exclude，可选）
    └── 排除组 1 [AND/OR 可切换]
        ├── 条件 F
        └── 条件 G
```

**计算逻辑**：`最终人群 = (组1 OR 组2 OR ...) - (排除组1 OR 排除组2 OR ...)`

### 2.2 逻辑层级

| 层级 | 逻辑关系 | 说明 | 行业对标 |
|------|----------|------|----------|
| 满足区 vs 排除区 | 差集（-） | 满足区命中 - 排除区命中 | 所有 CDP 产品 |
| 组间 | OR（固定） | 满足任一组即命中 | 神策、GrowingIO、Segment |
| 组内条件间 | AND / OR（可切换） | 用户可选「全部满足」或「任一满足」 | 神策（每组最多 10 条件）、Adobe AEP |

### 2.3 规则示例

**场景：续费召回用户（排除 VIP）**

```
满足：
  组1 [AND]：会员状态 = 在期 AND 到期时间 30天内 AND 没做过「登录」(近7天)
  OR
  组2 [AND]：会员状态 = 已过期 AND 到期时间 90天内

排除：
  组3 [OR]：客户等级 = S OR 属于「VIP专属服务」人群包
```

**场景：高价值活跃用户**

```
满足：
  组1 [AND]：身份标签 包含任一 [传术师, 航海家] AND 做过「评论」>=10次(近30天)

排除：
  组2 [AND]：好友状态 = 客户删除 AND 没做过「登录」(近90天)
```

## 3. 五种条件类型

### 3.1 属性条件（与 v2.0 一致）

基于用户静态属性筛选，维度注册制不变。

**格式**：`[维度] [操作符] [值]`

**示例**：`会员状态 等于 在期`

### 3.2 事件条件（新增，核心能力）

参考神策/Segment，基于用户行为事件筛选。

**格式**：`[做过/没做过] [事件名] [属性筛选] [次数条件] [时间窗口]`

**示例**：`做过 发帖 话题=短视频 >=5次 近30天`

#### 事件条件结构

| 要素 | 说明 | 示例 |
|------|------|------|
| 发生性（必填） | 做过 / 没做过 | 做过 |
| 事件名（必填） | 从注册事件中选择 | 发帖、评论、登录 |
| 属性筛选（可选） | 对事件的属性做过滤 | 话题 = 短视频 |
| 次数条件（做过时） | 大于/等于/小于/范围 | >= 5 次 / 1~10 次 |
| 聚合条件（可选） | 对事件属性做聚合 | 金额求和 >= 1000 元 |
| 时间窗口（必填） | 行为发生的时间范围 | 近 7 天 / 近 30 天 / 指定日期区间 |

#### 初始事件清单

| 事件代码 | 显示名 | 可筛选属性 | 数据源 |
|----------|--------|------------|--------|
| post | 发帖 | 话题、帖子类型 | 鱼丸系统 |
| comment | 评论 | — | 鱼丸系统 |
| read | 阅读 | 话题 | 鱼丸系统 |
| like | 点赞 | — | 鱼丸系统 |
| anchor | 投锚 | — | 鱼丸系统 |
| collect | 收藏 | — | 鱼丸系统 |
| join_voyage | 参加航海 | 航海名称 | 鱼丸系统 |
| login | 登录 | 登录方式 | 用户系统 |
| search | 搜索 | 搜索关键词 | 鱼丸系统 |
| live_watch | 观看直播 | 直播间名称 | 直播系统 |
| live_comment | 直播评论 | 直播间名称 | 直播系统 |
| order_pay | 支付订单 | 商品名称、金额 | 订单系统 |

> 事件注册与维度注册同理——后端配置中新增一个事件定义，前端自动可用。

### 3.3 人群包条件（新增）

支持基于已有人群包的包含/排除，实现人群包之间的运算。

**格式**：`[属于/不属于] [人群包名称]`

**示例**：`属于 活跃用户包`

#### 操作说明

| 操作 | 等价逻辑 | 场景 |
|------|----------|------|
| 属于人群包 A | 用户 IN A 的结果集 | 「在活跃用户中筛选高消费的」 |
| 不属于人群包 A | 用户 NOT IN A 的结果集 | 「排除已转化用户」 |

#### 组合运算（通过条件组合实现）

| 运算 | 实现方式 |
|------|----------|
| 交集 A ∩ B | 同一组内：属于 A AND 属于 B |
| 并集 A ∪ B | 不同组：属于 A（组1） OR 属于 B（组2） |
| 差集 A - B | 满足区放 A，排除区放 B |
| 复杂 (A ∩ B) - C | 满足区：属于 A AND 属于 B；排除区：属于 C |

### 3.4 标签条件（v4.0 新增）

基于 CDP 标签体系的标签值筛选，支持企微标签、CRM 标签、自动化标签三种来源。

**格式**：`[标签来源] [标签组] [操作符] [标签值]`

**示例**：`自动化标签 > 全局-生命周期 包含 流失期`

#### 标签来源

| 来源 | 说明 | 标签组数 |
|------|------|---------|
| 企微标签 | 企微侧边栏手动打标 | 4 组（转化/交付/运营/来源） |
| CRM 标签 | CRM 系统业务标签 | 4 组（销售阶段/客户价值/服务管理/运营分类） |
| 自动化标签 | CDP 标签引擎自动计算 | 9 组（全局 3 + 社区 2 + 航海 2 + 深海 2） |

#### 操作符

| 操作符 | 说明 | 示例 |
|--------|------|------|
| 包含 | 标签值包含指定值 | 生命周期标签 包含 "流失期" |
| 不包含 | 标签值不包含指定值 | 创作者等级 不包含 "PGC专业" |
| 等于 | 精确匹配 | 好友状态 等于 "客户删除" |
| 不等于 | 不匹配 | 续费状态 不等于 "已过期" |

> 标签条件支持三级联动：来源 → 标签组 → 标签值，前端动态加载。

### 3.5 评分条件（v4.0 新增）

基于 CDP 评分引擎的数值型评分筛选，支持健康分、续费倾向分、参与度分、CLV 分等。

**格式**：`[评分类型] [操作符] [分值]`

**示例**：`续费倾向分 小于 50`

#### 评分类型

| 评分 | 范围 | 说明 | 更新频率 |
|------|------|------|---------|
| 健康分 | 0-100 | 综合衡量客户当前状态 | 每日 |
| 续费倾向分 | 0-100 | 预测会员续费概率 | 每日 |
| 参与度分 | 0-100 | 衡量社群参与深度 | 每日 |
| CLV 分 | 0-100 | 客户终身价值预测 | 每月 |
| RFM-R 分 | 1-5 | Recency 最近互动 | 每日 |
| RFM-F 分 | 1-5 | Frequency 互动频率 | 每日 |
| RFM-M 分 | 1-5 | Monetary 综合价值 | 每日 |

#### 操作符

| 操作符 | 格式 | 示例 |
|--------|------|------|
| 大于 | > N | 健康分 > 80 |
| 大于等于 | >= N | 参与度分 >= 60 |
| 等于 | = N | RFM-R = 5 |
| 小于等于 | <= N | 续费倾向分 <= 30 |
| 小于 | < N | CLV 分 < 20 |
| 区间 | N1 ~ N2 | 健康分 40 ~ 79 |

> 评分条件特别适用于精细化运营场景——如「续费倾向分 < 50 且到期前 30 天」圈选续费风险用户。

## 4. 时间窗口体系

参考 Segment/Adobe，标准化时间窗口类型：

| 类型 | 含义 | 参数 | 示例 |
|------|------|------|------|
| within_days | 过去 N 天内 | days: number | 过去 7 天内 |
| beyond_days | 过去 N 天前 | days: number | 超过 30 天前 |
| between_dates | 日期区间 | start: date, end: date | 2026-01-01 ~ 2026-03-01 |
| before_date | 某日期之前 | date: date | 2026-03-01 之前 |
| after_date | 某日期之后 | date: date | 2026-03-01 之后 |
| today | 今天 | — | — |
| this_week | 本周 | — | — |
| this_month | 本月 | — | — |

**时间窗口用于**：
- 属性条件的日期类维度（注册时间、到期时间）
- 事件条件的发生时间（做过 X 事件 近 30 天内）

## 5. 数据结构

### 5.1 条件类型定义

```typescript
// ============ 条件类型 ============

// 属性条件
interface AttributeCondition {
  type: 'attribute';
  dimensionCode: string;      // 维度代码
  operator: string;           // 操作符代码
  value: any;                 // 值
}

// 事件条件
interface EventCondition {
  type: 'event';
  occurrence: 'did' | 'did_not';       // 做过 / 没做过
  eventCode: string;                    // 事件代码
  propertyFilters?: PropertyFilter[];   // 事件属性筛选（可选）
  countOperator?: string;               // 次数操作符（做过时）: gte, lte, eq, between
  countValue?: number | [number, number]; // 次数值
  aggregation?: {                       // 聚合条件（可选）
    property: string;                   // 聚合属性，如 "amount"
    function: 'sum' | 'avg' | 'max' | 'min' | 'count';
    operator: string;                   // gte, lte, eq, between
    value: number | [number, number];
  };
  timeWindow: TimeWindow;              // 时间窗口（必填）
}

// 事件属性筛选
interface PropertyFilter {
  property: string;     // 属性名
  operator: string;     // 操作符
  value: any;           // 值
}

// 人群包条件
interface AudienceCondition {
  type: 'audience';
  operator: 'in' | 'not_in';    // 属于 / 不属于
  audienceId: number;             // 人群包 ID
}

// 标签条件（v4.0 新增）
interface TagCondition {
  type: 'tag';
  tagSource: 'wecom' | 'crm' | 'auto';  // 标签来源
  tagGroup: string;                       // 标签组名
  operator: 'contains' | 'not_contains' | 'eq' | 'neq'; // 操作符
  tagValue: string;                       // 标签值
}

// 评分条件（v4.0 新增）
interface ScoreCondition {
  type: 'score';
  scoreType: 'health' | 'renewal' | 'engagement' | 'clv' | 'rfm_r' | 'rfm_f' | 'rfm_m';
  operator: 'gt' | 'gte' | 'eq' | 'lte' | 'lt' | 'between';
  value: number | [number, number];       // 单值或区间
}

// 条件联合类型（v4.0：3 → 5 种）
type Condition = AttributeCondition | EventCondition | AudienceCondition | TagCondition | ScoreCondition;
```

### 5.2 时间窗口定义

```typescript
type TimeWindow =
  | { type: 'within_days'; days: number }
  | { type: 'beyond_days'; days: number }
  | { type: 'between_dates'; start: string; end: string }
  | { type: 'before_date'; date: string }
  | { type: 'after_date'; date: string }
  | { type: 'today' }
  | { type: 'this_week' }
  | { type: 'this_month' };
```

### 5.3 规则结构定义

```typescript
// 条件组（组内逻辑可切换）
interface ConditionGroup {
  logic: 'and' | 'or';        // 组内条件间的逻辑关系
  conditions: Condition[];
}

// 人群包规则
interface AudienceRule {
  include: {                   // 满足条件（组间 OR）
    groups: ConditionGroup[];
  };
  exclude?: {                  // 排除条件（可选，组间 OR）
    groups: ConditionGroup[];
  };
}
```

```typescript
interface EventDefinition {
```


### 7.2 条件行模板

条件行第一列是**条件类型选择器**，切换后右侧动态渲染对应的输入组件：

**属性条件行**：
```
[用户属性 ▾] │ [维度 ▾] [操作符 ▾] [值 ▾/输入]  [×]
```

**事件条件行**：
```
[用户行为 ▾] │ [做过/没做过 ▾] [事件 ▾] [属性筛选...] [>=▾] [5] 次  [过去30天 ▾]  [×]
```

**人群包条件行**：
```
[人群包 ▾] │ [属于/不属于 ▾] [人群包选择 ▾]  [×]
```

### 7.3 组内 AND/OR 切换

每个条件组左上角有一个下拉：`[且(AND) ▾]` / `[或(OR) ▾]`

切换后，条件之间的连接词文字同步变化：
- AND 模式：条件间显示「且」
- OR 模式：条件间显示「或」

### 7.4 交互规则

| 操作 | 行为 |
|------|------|
| 添加条件 | 在当前组末尾新增一行，默认为属性条件 |
| 删除条件 | 点击行末 × 删除该条件；组内最后一个条件不可删除 |
| 添加条件组 | 在满足区末尾新增一个空条件组，组间显示「或（OR）」 |
| 删除条件组 | 点击组标题栏 × 删除整个组；至少保留一个满足组 |
| 添加排除 | 首次点击时显示排除区，新增一个空排除组 |
| 切换条件类型 | 切换第一列下拉后，该行右侧输入组件重新渲染 |
| 预估人数 | 点击按钮后异步计算，显示人数和计算时间 |
| 保存 | 校验规则完整性（每个条件的必填项），通过后保存 |

### 7.5 校验规则

| 校验项 | 规则 |
|--------|------|
| 名称 | 必填，最长 50 字符 |
| 条件完整性 | 每个条件行的必填字段不能为空 |
| 组非空 | 每个条件组至少有 1 个条件 |
| 满足区非空 | 至少有 1 个满足组 |
| 人群包引用 | 不能引用自身，不能循环引用 |

## 9. 非功能需求

| 项 | 要求 |
|----|------|
| 条件组上限 | 满足区最多 10 组，排除区最多 5 组 |
| 组内条件上限 | 每组最多 10 个条件（对齐神策） |
| 人群包总数上限 | 暂不限制（后续按套餐配额） |
| 嵌套引用深度 | 人群包条件引用链最深 3 层 |
| 预估计算超时 | 30 秒内未返回则提示「计算中，请稍后查看」 |
| 自动刷新频率 | 每日凌晨 2:00 自动计算 |
